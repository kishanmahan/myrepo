👉 Role = Permissions ka group (bundle)
Matlab:-

Iske andar:
Create User
Delete User
Modify Data

⚙️ 2️⃣ Role kya karta hai?
Role ka main kaam:-
✔️ 1. Access control simplify karna
Direct permission dene ki jagah role assign karte hain
✔️ 2. Security manage karna
Kaun kya access karega → role decide karta hai
✔️ 3. Automation
New employee aaye → role assign → automatically access mil gaya
✔️ 4. Governance (Audit)
Kaun kya use kar raha hai → easily track hota hai.

🔥 3️⃣ Inheritance Types (Deep Explanation):-
🔹 1. NONE:-
✔️ Meaning:-
Role independent hai
Kisi aur role se link nahi
✔️ Example:-
Role: Basic_User
→ No child / parent
✔️ Use:-
Simple roles
Direct assignment
-------------------------------------------------------------
🔹 2. PERMISSIVE (Most Important 🔥):-
✔️ Meaning:-
👉 Child role ke saare permissions parent me add ho jate hain
➡️ "ADD / UNION" logic

✔️ Formula:-
Final Access = Parent + Child
✔️ Example:-
Role A: Developer
Role B: Git Access
A includes B
👉 Result:-
Developer role me Git access automatically aa gaya.
----------------------------------------------------------------
🔹 3. RESTRICTIVE (Advanced 🔥):-
✔️ Meaning:-
👉 Child role ka access blindly add nahi hota
👉 Filter / restriction apply hoti hai

✔️ Formula:-
Final Access = Parent + (Filtered Child Access)
✔️ Example:
Role A: Manager
Role B: Finance Access

A includes B
BUT policy says:-
→ Manager only sees his department data
👉 Yaha full access nahi mila — restricted mila

✔️ Use Case:-
Data filtering
Security policies
Compliance (SOX, audit)

Business Role IT Roles ko include karta hai:---✅ Haan  
ka matlab kya?

🧠 Iska matlab kya hai?
🔹 1. “Direct access nahi deta” ❌
👉 Business Role ke andar:-
koi actual permission nahi hoti
koi system access directly defined nahi hota

📌 Example:-
Business Role: Developer

👉 Isme khud se:-
na DB access hai
na Git access hai
❌ Kuch bhi direct nahi.

🔹 2. “IT Roles ko include karta hai” ✅
👉 Business Role ke andar:-
multiple IT Roles add hote hain

📌 Example:-
Business Role: Developer

Includes:
   - IT Role: Git Access
   - IT Role: Jira Access
   - IT Role: DB Read
🔥 Ab actual magic yaha hota hai
👉 Jab user ko Business Role assign hota hai:-
User/developer → Business Role → IT Roles → Entitlements

👉 Result:-
User ko Git access mil gaya
User ko Jira access mil gaya
User ko DB access mil gaya

🧠 3️⃣ getPolicyViolations()/setPolicyViolations(List violations):-
✔️ Kya karta hai?
👉 Role ke saath linked policy violations return karta hai.
Policy Violations = rules ya policies jo role assign karne par break ho sakti hain
Example: SoD (Segregation of Duties) conflict

💻 Example:-
Bundle role = new Bundle();

List<String> violations = new ArrayList<>();
violations.add("SoD Conflict");
violations.add("Excessive Access");

role.setPolicyViolations(violations);

// Get violations
for(String v : role.getPolicyViolations()) {
    System.out.println(v);
}
📤 Output:-
SoD Conflict
Excessive Access

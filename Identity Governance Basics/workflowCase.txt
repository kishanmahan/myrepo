()SailPoint me WorkflowCase basically ek object hai jo workflow ke execution instance ko represent karta hai.
 Matlab jab bhi koi workflow trigger hota hai, ek WorkflowCase create hota hai, aur iske andar workflow ke sare state, 
 tasks, aur data track hote hain.WorkflowCase me work items, approvals, escalations store hote hain.

❗ Short Answer (direct):-
👉 WorkflowContext data leta hai → WorkflowCase se ✅
👉 Workflow se nahi ❌ kyuki workflow ke basis par workflowcase banata hai sailpoint.

🧠 Ab detail me samajh
🔷 1️⃣ Workflow kya hai?
Ye definition / design hai
XML/config hota hai
Static hota hai

👉 Example:-
OnboardingWorkflow.xml

👉 Isme:-
steps
transitions
rules

🔷 2️⃣ WorkflowCase kya hai?
Ye runtime instance hai
Jab workflow run hota hai tab banta hai
Isme actual data store hota hai
👉 Example:-
john.doe ka onboarding run → ek WorkflowCase banega
👉 Isme hota hai:-
arguments
status
messages
variables

🔷 3️⃣ WorkflowContext kya hai?
Ye execution helper / wrapper hai
WorkflowCase ko access karne ka tool

🔥 Relationship (sabse important):-
Workflow (design)
      ↓
WorkflowCase (runtime data)
      ↓
WorkflowContext (access / execution)
----------------------------------------------------------------------------------
()Detail me breakdown karte hain ki WorkflowCase ke andar kya kya hota hai:-

1️⃣ Basic Info / Metadata:-
getId() → WorkflowCase ka unique ID
getName() → Workflow ka naam
getStatus() → Workflow ka current status (Pending, Running, Completed, Failed)
getCreated() / getModified() → Case kab create aur last modified hua

2️⃣ Tasks / Steps:-
getTasks() → Is workflow ke andar kaun kaunse tasks hain
Har task me hota hai:-
Task ID
Task name
Assigned user / group
Status (Completed, Pending, Failed)

3️⃣ Input / Output Data:-
getInputData() → Workflow start hone se pehle jo data pass hua
getOutputData() → Workflow complete hone ke baad ka result
Example: Identity info, request details, approval decision

4️⃣ Associations:-
getIdentity() → Agar workflow kisi identity ke liye hai, to uska reference
getApplication() → Agar workflow kisi application provisioning ya deprovisioning se related hai
getScope() → Workflow ka scope (like access request, account, group)

5️⃣ Audit / Tracking:-
getLogs() → Workflow me kya kya actions perform hue, step by step
getComments() → Manual inputs ya approver comments
getExecutionTime() → Total time workflow execute hone me laga

6️⃣ State Management:-
Pending / Active / Completed / Failed / Cancelled
Ye status check karke workflow ko monitor aur manage kar sakte ho.

-------------------------------------
sare methods of workflowcase:-
📌 Basic Metadata / Identity:-
-------------------------------------
getId() → WorkflowCase unique ID
getName() → WorkflowCase ka name
getStatus() → Current status (e.g., Pending, Completed, Failed)
getType() → Workflow type (Approval / Provisioning etc.)
getCreated() → Case ka creation time
getModified() → Last modification time

📌 Workflow Definition & References:-
getWorkflow() → Associated Workflow definition object
getWorkflowName() → Workflow ka naam
getWorkflowVersion() → Version info

📌 Target / Scope:-
getTargetName() → Jiske liye workflow launch hua (like identity name)
getTargetType() → Kya target hai (Identity / Role / Plan)
getTarget() → Target object reference

📌 Input & Output:-
getInputData() → Workflow case ke input variables
getOutputData() → Result / output variables
getVariables() → All variables mapping

📌 State & Execution:-
getState() → Current internal state
getCurrentStep() → Step name jahan workflow currently hai
getStepStatus() → Step ka status
getExecutionTime() → Workflow ka run time
isCompleted() → Check if workflow finished
isFailed() → Check if workflow failed

📌 Tasks / TaskResults:-
getTasks() → List of tasks related to the case
getTaskResults() → TaskResult objects list
getPendingTasks() → Only pending tasks
getCompletedTasks() → Completed task list

📌 Approvals / WorkItems:-
getWorkItems() → Approval work items collection
getPendingWorkItems() → Pending approvals
getApprovedWorkItems() → Approved tasks
getWorkItemById() → Specific workitem

📌 Associations:-
getIdentity() → Identity object if identity workflow
getApplication() → Associated application object
getProvisioningPlan() → Provisioning plan object (if provisioning workflow)

📌 Audit / Logging:-
getLogs() → Workflow case logs
getMessages() → Messages added by workflow steps
addMessage(String msg) → Add message to case
addError(String err) → Add error message

📌 Status / Lifecycle:-
setStatus(String status) → Status update
markCompleted() → Mark workflow as Complete
markFailed() → Mark workflow as Failed

Security team approval

🔹 6. PolicyCheck
Policy violations check

🔹 7. BuildPlan
ProvisioningPlan banana

🔹 8. Provisioning
Account create / update

🔹 9. PostProvisioning
After actions (email, audit)

🔹 🔟 End / Complete
Workflow finish

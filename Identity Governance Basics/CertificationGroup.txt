()CertificationGroup kya hai?
CertificationGroup ek object hai jo multiple certifications ko ek group me organize karta hai.
👉 Matlab:-
Ek hi type ke ya related certifications ko group karke manage karna,
Review process ko easy aur structured banana.

🔹 Ye kya karta hai?
CertificationGroup ka main kaam:-
()Certifications ko group karna
Jaise: sabhi “Manager Reviews” ek group me

()Bulk handling:-
Ek saath multiple certifications ko process kar sakte ho.

()Tracking & management:-
Kaun sa certification kis group me hai → easily track.

🔹 Iske andar kya hota hai (Fields / Attributes)?
Typical fields 👇
1️⃣ Basic Info:-
id → unique ID
name → group ka naam

2️⃣ Certifications list:-
List<Certification> → group ke andar jitne certifications hain

3️⃣ Type:-
Kis type ka group hai
(Manager, Application Owner, Role-based, etc.)

4️⃣ Status:-
Pending / Completed / In Progress

5️⃣ Owner / Reviewer:-
Kaun group ko handle kar raha hai

6️⃣ Dates:-
Created date
Due date

🔹 Simple Example:-
CertificationGroup group = new CertificationGroup();

group.setName("Manager Certification Group");

List<Certification> certs = new ArrayList<>();
certs.add(cert1);
certs.add(cert2);

group.setCertifications(certs);

System.out.println(group.getName());
System.out.println(group.getCertifications().size());

👉 Output:-
Manager Certification Group
2

🔹 Kya ye DB me store hota hai?
👉 Haan bhai ✔️
Ye object database me persist hota hai
SailPoint (IdentityIQ) isko table me store karta hai
👉 Generally:-
CertificationGroup ka record DB me hota hai
Iske andar jo certifications hain → unka relation bhi store hota hai.

🔹 getOwner() me “Owner” kaun hota hai?
👉 Owner = wo banda (identity) jo CertificationGroup ka zimmedar hai
Matlab:-
Jo us group ko manage kare
Jo uska review/process control kare
Jo ensure kare ki certification complete ho

🔹 Real me Owner kaun ho sakta hai?
Situation pe depend karta hai 👇

1️⃣ Manager:-
👉 Agar Manager Certification hai
➡️ Owner = Manager
Example:-
User: Ravi
Manager: Amit
👉 Owner = Amit

2️⃣ Application Owner:-
👉 Agar Application access review hai
➡️ Owner = Application ka owner
Example:-
App: SAP
Owner: Rahul
👉 Owner = Rahul

3️⃣ Role Owner:-
👉 Agar role-based certification hai
➡️ Owner = Role ka owner

4️⃣ Admin / Governance Team:-
👉 Kabhi kabhi system assign karta hai
➡️ Owner = Admin ya Compliance team.

()Authentication kya hai?
--1️⃣ Authentication (()Authentication)
Meaning: Ye check karta hai “Who you are?”
Purpose: System ye verify karta hai ki user jo login kar raha hai, wo asli me wahi hai jo wo claim kar raha hai.
Methods:-
Password

Biometric (Fingerprint, Face ID)

()Authorization kya hai?
--Meaning: Ye check karta hai “What you can do?”
Purpose: System decide karta hai ki authenticated user ko kaun kaunse resources/permissions allowed hain.

Methods:-
Roles (Finance Manager, IT Admin)
Policies (Access policies, Approval workflows)
Groups & Permissions

Example:-
Rahul successfully login ho gaya (Authentication pass)

Ab system check karta hai:-
Rahul ko SAP me “Finance Manager” access hai?
Salesforce me read-only ya full access hai?
✅ Agar allowed → Action permit

()RBAC (Role Based Access Control) kya hai?
--1️⃣ RBAC kya hai?
Meaning: User ka access directly permissions se nahi, balki roles ke through assign hota hai.
Purpose: Access management ko easy, scalable, aur secure banane ke liye.
Core Idea:
Roles define karte hain permissions ka group
Users ko roles assign kiye jaate hain
System automatically user ke liye permissions apply karta hai.

2️⃣ Example Table:-
Role	              Permissions	                       Assigned Users
Finance Manager----SAP Full Access, Read Finance Reports----Rahul, Priya
IT Admin-----------Server Access, Admin Dashboard-----------Anil, Ravi
Employee-----------Email Access, Leave Portal---------------All Employees

Flow:-
Rahul ko “Finance Manager” role assign → Automatically SAP + Finance Reports access mil gaya
Agar Rahul role change hota hai → Access automatically update ho jata hai.

()3️⃣ RBAC Advantages:-
Easy Management: Directly har user ko permission assign nahi karna
Consistency: Same role wale sab users ka access same hota hai
Audit & Compliance: Easy to track who has access to what
Dynamic: Role change = Access change automatically.

4️⃣ Simple Code Example (SailPoint Style)
// Define Role
Role financeManager = new Role("Finance Manager");
financeManager.addPermission("SAP Full Access");
financeManager.addPermission("Finance Reports Read");

// Assign User to Role

// Check Access
if(rahul.hasPermission("SAP Full Access")){
    System.out.println("Access Granted!");
}else{
    System.out.println("Access Denied!");
}

import sailpoint.api.SailPointContext;
import sailpoint.object.Identity;
import sailpoint.object.Role;
import sailpoint.object.Application;
import sailpoint.object.ProvisioningPlan;
import sailpoint.object.ProvisioningPlan.AccountRequest;
import sailpoint.object.ProvisioningPlan.AttributeRequest;

Identity identity = context.getObjectByName(Identity.class, "rahul");

Role role = context.getObjectByName(Role.class, "FinanceRole");

Application app = context.getObjectByName(Application.class, "FinanceApp");

if(identity != null && role != null){

    // Assign role to user
    identity.add(role);

    // Create provisioning plan
    ProvisioningPlan plan = new ProvisioningPlan();
    plan.setIdentity(identity);

    // Account request
    AccountRequest accReq = new AccountRequest();
    accReq.setApplication(app.getName());
    accReq.setOperation(AccountRequest.Operation.Create);

    // Entitlement request
    AttributeRequest entitlement = new AttributeRequest();
    entitlement.setName("group");
    entitlement.setOperation(ProvisioningPlan.Operation.Add);
    entitlement.setValue("FinanceGroup");

    accReq.add(entitlement);

    plan.add(accReq);

    context.saveObject(identity);
    context.commitTransaction();

}

()1️⃣ Fine-Grained Access:-
Meaning: Access control itna detailed hai ki har chhoti permission ko control kiya ja sakta hai.
Example:-
Rahul ko Finance folder me read access, lekin write access sirf Manager ko.
Employee ko leave portal access, lekin salary details HR ko hi dikhe.

Meaning: Access real-time conditions aur attributes ke basis pe decide hota hai, na ki static roles se.
Example:-
Rahul Finance system me sirf office hours (9am–6pm) me access le sakta hai.
Agar Rahul VPN ya unknown device se login karta → Access deny.
Ek line me: Access automatically change hota hai based on rules or conditions.

Meaning: Resource ka criticality/importance level define karta hai.
Example:-
Resource	             Sensitivity Level	    Access Allowed
Public HR Policies-----------Low-----------------All Employees
Payroll Data-----------------High----------------HR + Finance Manager
Top Secret Financial Report--Very High-----------CFO only
SailPoint me ABAC policy me use hota hai, taaki only authorized users hi sensitive data access kare.

()4️⃣ Time of Day:-
Meaning: Access sirf specified time window me allow hota hai.
Example:-
Rahul ka SAP access: 9am – 6pm
Payroll reports: 10am – 4pm
After hours: Access denied

import sailpoint.api.SailPointContext;
import sailpoint.object.Identity;
import sailpoint.object.Link;
import sailpoint.object.Application;
import sailpoint.object.ProvisioningPlan;
import sailpoint.object.ProvisioningPlan.AccountRequest;
import sailpoint.object.ProvisioningPlan.AttributeRequest;
import sailpoint.object.ProvisioningPlan.Operation;

import java.util.List;

public class LeastPrivilegeExample {

    public static void applyLeastPrivilege(SailPointContext context) throws Exception {

        // Identity fetch
        Identity user = context.getObjectByName(Identity.class, "john.doe");

        // Application fetch
        Application app = context.getObjectByName(Application.class, "ActiveDirectory");

        // User ke accounts
        List<Link> links = user.getLinks();

        // Provisioning Plan
        ProvisioningPlan plan = new ProvisioningPlan();
        plan.setIdentity(user);

        for(Link link : links){

            if(link.getApplicationName().equals("ActiveDirectory")){

                // Example: Remove extra entitlement
                AccountRequest accReq = new AccountRequest();
                accReq.setApplication(app.getName());
                accReq.setNativeIdentity(link.getNativeIdentity());
                accReq.setOperation(Operation.Modify);

                AttributeRequest attrReq = new AttributeRequest();
                attrReq.setName("memberOf");
                attrReq.setOperation(Operation.Remove);
                attrReq.setValue("AdminGroup");

                accReq.add(attrReq);

                plan.add(accReq);

            }

        }

        // Save provisioning plan
        context.saveObject(plan);
        context.commitTransaction();

        System.out.println("Least Privilege applied for user: " + user.getName());
    }
}

import java.util.ArrayList;
import java.util.Date;
import java.util.List;

class Permission {

    String name;

    Permission(String name) {
        this.name = name;
    }
}

class PrivilegedRole {

    String roleName;
    List<Permission> permissions = new ArrayList<Permission>();

    PrivilegedRole(String roleName) {
        this.roleName = roleName;
    }

    void addPermission(Permission p) {
        permissions.add(p);
    }
}

class Identity {

    String name;
    boolean mfaVerified;

    Identity(String name) {
        this.name = name;
    }

    void verifyMFA() {
        mfaVerified = true;
        System.out.println("MFA verified for " + name);
    }
}

class PrivilegedAccessGrant {

    Identity user;
    PrivilegedRole role;
    Date start;
    Date end;

    PrivilegedAccessGrant(Identity user, PrivilegedRole role, Date start, Date end) {
        this.user = user;
        this.role = role;
        this.start = start;
        this.end = end;
    }

    void checkAccess() {

        Date today = new Date();

        if (!user.mfaVerified) {
            System.out.println("Access Denied: MFA not verified");
            return;
        }

        if (today.after(start) && today.before(end)) {

            System.out.println("Privileged Access Granted to " + user.name);

            for (Permission p : role.permissions) {
                System.out.println("Permission: " + p.name);
            }

        } else {
            System.out.println("Privileged Access Expired");
        }
    }
}

public class PAMExample {

    public static void main(String[] args) {

        Identity rahul = new Identity("Rahul");

        PrivilegedRole dbAdmin = new PrivilegedRole("Database Admin");

        dbAdmin.addPermission(new Permission("Read Database"));
        dbAdmin.addPermission(new Permission("Modify Database"));

        rahul.verifyMFA();

        Date end = new Date(System.currentTimeMillis() + (2 * 60 * 60 * 1000));

        grant.checkAccess();

    }

import java.util.Random;
import java.util.Scanner;
import java.io.OutputStream;
import java.net.HttpURLConnection;
import java.net.URL;

class Identity {

    String name;
    String mobileNumber;
    String systemOtp;
    boolean mfaVerified = false;

    Identity(String name, String mobileNumber) {
        this.name = name;
        this.mobileNumber = mobileNumber;
    }

    void sendOTP() {

        systemOtp = generateOTP();

        SmsService.sendSMS(mobileNumber, systemOtp);

        System.out.println("OTP sent to mobile number: " + mobileNumber);
    }

    void verifyMFA(Scanner scanner) {

        System.out.print("Enter OTP for " + name + ": ");
        String enteredOtp = scanner.nextLine();

        if(systemOtp.equals(enteredOtp)) {
            mfaVerified = true;
            System.out.println("MFA verified for " + name);
        } else {
            System.out.println("Invalid OTP for " + name);
        }
    }

    String generateOTP() {

        Random r = new Random();
        int otp = 100000 + r.nextInt(900000);

        return String.valueOf(otp);
    }
}

class SmsService {

    static void sendSMS(String mobile, String otp) {

        try {

            String apiKey = "YOUR_API_KEY";

            String message = "Your OTP is " + otp;

            String data = "authkey=" + apiKey +
                    "&mobiles=" + mobile +
                    "&message=" + message +
                    "&sender=OTPMSG" +
                    "&route=4";

            URL url = new URL("https://api.msg91.com/api/sendhttp.php");

            HttpURLConnection conn = (HttpURLConnection) url.openConnection();

            conn.setRequestMethod("POST");
            conn.setDoOutput(true);

            OutputStream os = conn.getOutputStream();
            os.write(data.getBytes());
            os.flush();
            os.close();

            int responseCode = conn.getResponseCode();

            System.out.println("SMS API Response Code: " + responseCode);

        } catch(Exception e) {
            e.printStackTrace();
        }
    }
}

import java.util.ArrayList;
import java.util.Date;
import java.util.List;

class Permission {

    String name;

    Permission(String name) {
        this.name = name;
    }
}

class Role {

    String roleName;
    List<Permission> permissions = new ArrayList<Permission>();

    Role(String roleName) {
        this.roleName = roleName;
    }

    void addPermission(Permission p) {
        permissions.add(p);
    }
}

class AccessGrant {

    Role role;
    Date startDate;
    Date endDate;

    AccessGrant(Role role, Date startDate, Date endDate) {
        this.role = role;
        this.startDate = startDate;
        this.endDate = endDate;
    }

    boolean isActive() {

        Date today = new Date();

        return today.after(startDate) && today.before(endDate);
    }
}

class Identity {

    String name;
    List<AccessGrant> grants = new ArrayList<AccessGrant>();

    Identity(String name) {
        this.name = name;
    }

    void grantRole(Role role, Date start, Date end) {

        AccessGrant grant = new AccessGrant(role, start, end);
        grants.add(grant);

        System.out.println("Role " + role.roleName + " granted to " + name +
                " from " + start + " to " + end);
    }

    void checkAccess() {

        System.out.println("Checking active permissions for " + name);

        for (AccessGrant g : grants) {

            if (g.isActive()) {

                for (Permission p : g.role.permissions) {
                    System.out.println("Active Permission: " + p.name);
                }

            } else {
                System.out.println("Access expired for role: " + g.role.roleName);
            }
        }
    }
}

public class TimeBoundAccessExample {

    public static void main(String[] args) {

        Role financeRole = new Role("Finance Role");

        financeRole.addPermission(new Permission("Read Finance Reports"));
        financeRole.addPermission(new Permission("Write Finance Reports"));

        Identity rahul = new Identity("Rahul");

        Date start = new Date(); // now

        Date end = new Date(System.currentTimeMillis() + (3 * 24 * 60 * 60 * 1000));
        // 3 days access

        rahul.grantRole(financeRole, start, end);

        rahul.checkAccess();
    }
}

import java.util.ArrayList;
import java.util.List;

class Role {

    String roleName;
    List<String> permissions = new ArrayList<String>();
    List<Role> childRoles = new ArrayList<Role>();

    Role(String roleName) {
        this.roleName = roleName;
    }

    void addPermission(String permission) {
        permissions.add(permission);
    }

    void addChildRole(Role role) {
        childRoles.add(role);
    }

    List<String> getAllPermissions() {

        List<String> allPermissions = new ArrayList<String>();

        allPermissions.addAll(permissions);

        for (Role child : childRoles) {
            allPermissions.addAll(child.getAllPermissions());
        }

        return allPermissions;
    }
}

class Identity {

    String name;
    List<Role> roles = new ArrayList<Role>();

    Identity(String name) {
        this.name = name;
    }

    void assignRole(Role role) {
        roles.add(role);
    }

    List<String> getPermissions() {

        List<String> allPermissions = new ArrayList<String>();

        for (Role role : roles) {
            allPermissions.addAll(role.getAllPermissions());
        }

        return allPermissions;
    }
}

import java.util.ArrayList;
import java.util.List;

import java.util.HashMap;

class Identity {

class Role {

        this.roleName = roleName;

public class RoleMiningExample {

    public static void main(String[] args) {

        // Create Users
        Identity rahul = new Identity("Rahul");
        rahul.addPermission("Read Finance");
        rahul.addPermission("Write Finance");

        Identity priya = new Identity("Priya");
        priya.addPermission("Read Finance");
        priya.addPermission("Write Finance");

        Identity anil = new Identity("Anil");
        anil.addPermission("View HR");
        anil.addPermission("Edit HR");

        Identity riya = new Identity("Riya");
        riya.addPermission("View HR");
        riya.addPermission("Edit HR");

        List<Identity> users = new ArrayList<Identity>();
        users.add(rahul);
        users.add(priya);
        users.add(anil);
        users.add(riya);

        // Role Mining
        Map<String, Role> discoveredRoles = new HashMap<String, Role>();

        for (Identity user : users) {

            boolean roleFound = false;

            for (Role role : discoveredRoles.values()) {
                if (role.permissions.equals(user.permissions)) {
                    System.out.println(user.name + " assigned to existing role: " + role.roleName);
                    roleFound = true;
                    break;
                }
            }

            if (!roleFound) {
                String roleName = "Role_" + (discoveredRoles.size() + 1);

                Role newRole = new Role(roleName);
                newRole.permissions.addAll(user.permissions);

                discoveredRoles.put(roleName, newRole);

                System.out.println(user.name + " assigned to new role: " + roleName);
            }
        }

        System.out.println("\nDiscovered Roles:");

        for (String rName : discoveredRoles.keySet()) {
            System.out.println(rName + " -> Permissions: " + discoveredRoles.get(rName).permissions);
        }
    }
}

()Role Lifecycle kya hai?
--Role Lifecycle IAM (Identity and Access Management) me ek process hota hai jisme role ko create,
manage, update aur retire kiya jata hai.
Matlab ek role ka poora life cycle — jab role banaya jata hai tab se lekar jab tak usko 
remove/retire nahi kar diya jata. 🔐

Role Lifecycle ke main stages:-
1️⃣ Role Creation:-
Sabse pehle role create kiya jata hai.
Example:-
Company me Finance Analyst job hai.
To role banega:-
Role Name: Finance Analyst
Permissions:
- Read Finance Data
- Write Finance Reports

2️⃣ Role Assignment:-
Role ko users ko assign kiya jata hai.
Example:-
Rahul → Finance Analyst Role
Priya → Finance Analyst Role
Ab Rahul aur Priya ko automatically same permissions mil jayenge.

3️⃣ Role Certification / Review:-
Time-to-time manager ya security team review karti hai ki role sahi hai ya nahi.
Questions:-
Kya role me extra permissions hain?
Kya user ko abhi bhi role chahiye?
Kya compliance rules follow ho rahe hain?
Example:-
Manager review karta hai:-
Rahul ko Finance Analyst role abhi bhi chahiye? ✔

4️⃣ Role Modification:-
Agar job change ho ya policy change ho to role update kiya jata hai.
Example:-
Pehle permissions:-
Read Finance
Write Finance

Update ke baad:-
Read Finance
Write Finance
Approve Finance

5️⃣ Role De-assignment:-
Agar user job change kare ya company leave kare to role remove kar diya jata hai.
Example:-
Rahul HR department me shift ho gaya
→ Finance Analyst role remove

6️⃣ Role Retirement:-
Agar role ab company me use nahi ho raha to role retire/delete kar diya jata hai.
Example:-
Old Role: Legacy Finance Role
→ Retired

import java.util.ArrayList;
import java.util.List;

class Permission {

    String name;

    Permission(String name) {
        this.name = name;
    }
}

class TechnicalRole {

    String roleName;
    String application;
    List<Permission> permissions = new ArrayList<Permission>();

    TechnicalRole(String roleName, String application) {
        this.roleName = roleName;
        this.application = application;
    }

    void addPermission(Permission p) {
        permissions.add(p);
    }
}

class BusinessRole {

    String roleName;
    String status = "ACTIVE";
    List<TechnicalRole> technicalRoles = new ArrayList<TechnicalRole>();

    BusinessRole(String roleName) {
        this.roleName = roleName;
    }

    void addTechnicalRole(TechnicalRole role) {
        technicalRoles.add(role);
    }

    void retireRole() {
        status = "RETIRED";
        System.out.println("Role retired: " + roleName);
    }
}

class Identity {

    String userId;
    List<BusinessRole> roles = new ArrayList<BusinessRole>();

    Identity(String userId) {
        this.userId = userId;
    }

    void assignRole(BusinessRole role) {

        if(role.status.equals("RETIRED")) {
            System.out.println("Cannot assign retired role");
            return;
        }

        roles.add(role);
        System.out.println("Role " + role.roleName + " assigned to " + userId);

        applyPermissions(role);
    }

    void applyPermissions(BusinessRole role) {

        for(TechnicalRole t : role.technicalRoles) {

            System.out.println("Application: " + t.application);

            for(Permission p : t.permissions) {
                System.out.println("Permission granted: " + p.name);
            }
        }
    }
}

class PolicyEngine {

    static void checkSoD(BusinessRole role) {

        boolean createVendor = false;
        boolean approvePayment = false;

        for(TechnicalRole t : role.technicalRoles) {
            for(Permission p : t.permissions) {

                if(p.name.equals("Create Vendor")) {
                    createVendor = true;
                }

                if(p.name.equals("Approve Payment")) {
                    approvePayment = true;
                }
            }
        }

        if(createVendor && approvePayment) {
            System.out.println("SoD Violation detected in role: " + role.roleName);
        }
    }
}

    static void log(String action) {
        System.out.println("AUDIT LOG: " + action);
    }
}

()Role Risk kya hai?
Role Risk = kisi role ke permissions ki wajah se security ya compliance risk.

Example:-
Example 1 (High Risk Role)
Role: Finance Admin
Permissions:
- Read Financial Data
- Modify Financial Data
- Delete Financial Records

Yeh role high risk hai kyunki user:-
data dekh sakta hai
data change kar sakta hai

1️⃣ Sensitive Permissions:-
Example:-
Delete Database
Approve Payments

Agar ek hi role me conflicting permissions ho.
Example:-
Create Vendor

Ye dangerous hai kyunki ek hi person fraud kar sakta hai.

3️⃣ Excess Privileges:-
Role me zarurat se zyada permissions ho.
Example:-
Developer Role:
- Read Code
- Deploy Code
- Delete Production Database

import java.util.ArrayList;
import java.util.List;

class TechnicalRole {

    String roleName;
    List<String> permissions = new ArrayList<String>();

    TechnicalRole(String roleName) {
        this.roleName = roleName;
    }

    void addPermission(String permission) {
        permissions.add(permission);
    }

    int calculateRisk() {

        int riskScore = 0;

        for (String perm : permissions) {

            if (perm.equals("Delete Database")) {
                riskScore += 50;
            }

            if (perm.equals("Modify Finance Data")) {
                riskScore += 30;
            }

            if (perm.equals("Read Finance Reports")) {
                riskScore += 10;
            }
        }

        return riskScore;
    }
}

public class RoleRiskExample {

    public static void main(String[] args) {

        TechnicalRole adminRole = new TechnicalRole("System Admin");

        adminRole.addPermission("Read Finance Reports");
        adminRole.addPermission("Modify Finance Data");
        adminRole.addPermission("Delete Database");

        int risk = adminRole.calculateRisk();

        System.out.println("Role: " + adminRole.roleName);
        System.out.println("Risk Score: " + risk);

        if (risk > 60) {
            System.out.println("High Risk Role!");
        } else {
            System.out.println("Low / Medium Risk Role");
        }
    }
}

                    hasRead = true;
                }

                if (perm.equals("Write Finance Reports")) {
                    hasWrite = true;
                }
            }
        }

            System.out.println("SoD Warning: User has both Read and Write Finance access.");
        }

    }

SailPoint में mainly 3 types की policies होती हैं:

1️⃣ Segregation of Duties (SoD):-
दो conflicting permissions एक user को नहीं मिलनी चाहिए।

अगर एक ही user को दोनों मिल गए → SoD violation.

Example activities:-

Sensitive data access
Privilege change
अगर activity suspicious है → policy violation।

import sailpoint.api.SailPointContext;
import sailpoint.object.Identity;
import sailpoint.object.Link;
import sailpoint.tools.GeneralException;

import java.util.List;

public class SODPolicyCheck {

    private SailPointContext context;

    public SODPolicyCheck(SailPointContext context) {
        this.context = context;
    }

    public void checkSODViolation(String identityName) throws GeneralException {

        Identity user = context.getObjectByName(Identity.class, identityName);

        if(user == null) {
            System.out.println("User not found");
            return;
        }

        boolean hasCreatePayment = false;
        boolean hasApprovePayment = false;

        List<Link> accounts = user.getLinks();

        for(Link link : accounts) {

            List entitlements = link.getEntitlements();

            for(Object ent : entitlements) {

                String entitlement = ent.toString();

                if(entitlement.equalsIgnoreCase("Create_Payment")) {
                    hasCreatePayment = true;
                }

                if(entitlement.equalsIgnoreCase("Approve_Payment")) {
                    hasApprovePayment = true;
                }

            }

        }

        if(hasCreatePayment && hasApprovePayment) {

            System.out.println("SOD POLICY VIOLATION DETECTED");
            System.out.println("User: " + user.getName());
            System.out.println("Conflicting Access:");
            System.out.println("Create_Payment");
            System.out.println("Approve_Payment");

        } else {

            System.out.println("No SOD violation for user: " + user.getName());

        }

    }

}

import java.util.HashMap;
import java.util.Map;

public class ActivityPolicyEngine {

    private static Map<String, Integer> downloadTracker = new HashMap<>();

    private static final int DOWNLOAD_LIMIT = 100;

    public static void recordDownload(String user) {

        int count = downloadTracker.getOrDefault(user, 0);

        count++;

        downloadTracker.put(user, count);

        checkPolicy(user, count);
    }

    private static void checkPolicy(String user, int count) {

        if(count > DOWNLOAD_LIMIT) {

            System.out.println("ACTIVITY POLICY VIOLATION DETECTED");
            System.out.println("User: " + user);
            System.out.println("Downloaded Files: " + count);
            System.out.println("Limit: " + DOWNLOAD_LIMIT);
            System.out.println("Action: Security Alert Generated");

        } else {

            System.out.println("Download Allowed for user: " + user +
                    " | Total Downloads: " + count);

        }
    }
}

public class RiskPolicyEngine {

    private static final int RISK_THRESHOLD = 70;

    public static void evaluateRisk(Identity identity) {

        int risk = identity.getRiskScore();

        if (risk > RISK_THRESHOLD) {

            System.out.println("RISK POLICY VIOLATION");
            System.out.println("User: " + identity.getName());
            System.out.println("Role: " + identity.getRole());
            System.out.println("Risk Score: " + risk);
            System.out.println("Action: Access Blocked / Security Review");

        } else {

            System.out.println("Access Allowed");
            System.out.println("User: " + identity.getName());
            System.out.println("Risk Score: " + risk);

        }
    }
}

4️⃣ Password Policy Engine (Enterprise Style)
public class PasswordPolicyEngine {

    public static void validatePassword(String password) {

        boolean hasUpper = false;
        boolean hasLower = false;
        boolean hasDigit = false;
        boolean hasSpecial = false;

        if(password.length() < 8) {
            System.out.println("Policy Violation: Password length must be at least 8");
            return;
        }

        for(char c : password.toCharArray()) {

            if(Character.isUpperCase(c))
                hasUpper = true;

            else if(Character.isLowerCase(c))
                hasLower = true;

            else if(Character.isDigit(c))
                hasDigit = true;

            else
                hasSpecial = true;
        }

        if(!hasUpper) {
            System.out.println("Policy Violation: Missing uppercase letter");
            return;
        }

        if(!hasLower) {
            System.out.println("Policy Violation: Missing lowercase letter");
            return;
        }

        if(!hasDigit) {
            System.out.println("Policy Violation: Missing number");
            return;
        }

        if(!hasSpecial) {
            System.out.println("Policy Violation: Missing special character");
            return;
        }

        System.out.println("Password Accepted (Strong Password)");

    }
}

import java.util.ArrayList;
import java.util.List;

class Violation {
    private String user;
    private String policyName;
    private String reason;

    public Violation(String user, String policyName, String reason) {
        this.user = user;
        this.policyName = policyName;
        this.reason = reason;
    }

    public void printViolation() {
        System.out.println("VIOLATION DETECTED");
        System.out.println("User: " + user);
        System.out.println("Policy: " + policyName);
        System.out.println("Reason: " + reason);
        System.out.println("----------------------");
    }
}

public class ViolationDetectionEngine {

    private List<Violation> violations = new ArrayList<>();

    // SoD Example Check
    public void checkSoD(String user, boolean hasCreate, boolean hasApprove) {
        if(hasCreate && hasApprove) {
            violations.add(new Violation(user, "SoD Policy", "User has conflicting roles: Create & Approve Payment"));
        }
    }

    // Activity Example Check
    public void checkActivity(String user, int downloads, int limit) {
        if(downloads > limit) {
            violations.add(new Violation(user, "Activity Policy", "Downloaded " + downloads + " files exceeding limit " + limit));
        }
    }

    // Identity Attribute Example Check
    public void checkAttribute(String user, String department, String system) {
        if(department.equals("Finance") && system.equals("HRSystem")) {
            violations.add(new Violation(user, "Attribute Policy", "Finance users cannot access HR System"));
        }
    }

    public void printAllViolations() {
        if(violations.isEmpty()) {
            System.out.println("No violations detected");
        } else {
            for(Violation v : violations) {
                v.printViolation();
            }
        }
    }
}
5️⃣ Main Program
public class ViolationMain {

    public static void main(String[] args) {

        ViolationDetectionEngine engine = new ViolationDetectionEngine();

        // SoD Violation
        engine.checkSoD("Rahul", true, true);

        // Activity Violation
        engine.checkActivity("Amit", 120, 100);

        // Identity Attribute Violation
        engine.checkAttribute("John", "Finance", "HRSystem");

        // Print all violations
        engine.printAllViolations();
    }
}

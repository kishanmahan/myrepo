1️⃣ Application kya hai?
Application app
Kya hai: Ye ek target system ko represent karta hai jaha identity ka account create hota hai (Active Directory, SAP, Jira, etc.).
Kya karta hai:-
Identity ke accounts ke liye provisioning define karta hai.

👉 Application = external system jisme user ka account hota hai.
Matlab:-
SailPoint ek central system hai
Aur Application = target system (jaha access dena hai)
✅ Example:-
Application	           Meaning
Active Directory:---Windows login system
SAP	ERP:------------system
Salesforce:----------CRM system
Database:-----------DB access system
➡ Ye sab Application objects hain.

🔹 2️⃣ Ye kya karta hai?
👉 Ye define karta hai:-
system ka connection
user accounts ka structure
provisioning ka flow

🔥 Use:-
account create / delete
password reset
access assign

🔹 3️⃣ Kya ye DB me store hota hai?
✅ Haan bhai, DB me store hota hai.
📦 Structure:-
Application (SAP)
   ↓
Accounts
   ↓
Entitlements

🔹 4️⃣ Iske andar kya hota hai?
Field	     Meaning
name:--------application ka naam
type:--------system type (LDAP, DB, etc.)
connector:---kaise connect karega
attributes:--config values
schemas:-----account structure
owner:-------responsible person

🔹 5️⃣ Important Methods:-
🔸 Basic:-
getName() → application ka naam deta hai
setName(String name) → naam set karta hai
🔸 Type:-
getType() → application type deta hai
setType(String type) → type set karta hai
🔸 Connector:-
getConnector() → connector deta hai
setConnector(String connector) → connector set karta hai
🔸 Attributes:-
getAttribute(String name) → config value deta hai
setAttribute(String name, Object value) → config set karta hai
🔸 Schema:-
getSchemas() → account structure deta hai
setSchemas(Object schemas) → schema set karta hai
🔸 Owner:-
getOwner() → owner deta hai
setOwner(String owner) → owner set karta hai
🔸 Utility:-
toString() → readable string
equals(Object obj) → compare karta hai
hashCode() → hash generate karta hai

🚀 🔥 Full Code (SailPointContext ke saath):-
import sailpoint.api.SailPointContext;
import sailpoint.api.SailPointFactory;
import sailpoint.object.Application;

public class ApplicationExample {

    public static void main(String[] args) throws Exception {

        SailPointContext context = SailPointFactory.getCurrentContext();

        Application app = new Application();

        // Basic
        app.setName("SAP");

        // Type
        app.setType("ERP");

        // Connector
        app.setConnector("SAP Connector");

        // Attributes
        app.setAttribute("url", "https://sap.company.com");

        // Owner
        app.setOwner("IT Admin");

        // Print
        System.out.println(app.getName());
        System.out.println(app.getType());
        System.out.println(app.getConnector());
        System.out.println(app.getAttribute("url"));
        System.out.println(app.getOwner());

        // Save
        context.saveObject(app);
        context.commitTransaction();

        System.out.println("✅ Application Saved");
    }
}
📤 Output:-
SAP
ERP
SAP Connector
https://sap.company.com
IT Admin
✅ Application Saved

🔥 2️⃣ ERP Application (DETAILED 🔥):-
👉 Example: SAP,Oracle ERP

✅ ERP Application kya karta hai?
👉 Ye business operations ka access control karta hai
Matlab:-
kaun finance data dekh sakta hai
kaun salary access karega
kaun invoice approve karega

🔥 Internal kaam (deep samajh 🔥):-
🔹 1. Role Based Access deta hai
👉 Direct permission nahi deta
👉 Roles ke through deta hai
Example:-
FI_USER → invoice dekh sakta hai
HR_USER → employee data dekh sakta hai

🔹 2. Business Process Control karta hai:-
👉 ERP me sirf access nahi hota, process control hota hai
Example:-
purchase order create
payment approve
salary process

🔹 3. Risk & Compliance check karta hai 🔥
👉 Sabse important part
Example:-
Admin + Auditor ek saath ❌
Create + Approve same banda ❌
👉 Agar violation → alert 🚨

🔹 4. Certification (Review) hota hai:-
👉 manager check karta hai:-
Rahul ko ye access chahiye ya nahi

🎯 Real Scenario:-
👨 Rahul finance team me hai:-
SAP role mila → FI_USER
wo invoice dekh sakta hai
agar extra role mila → violation
certification me review hoga

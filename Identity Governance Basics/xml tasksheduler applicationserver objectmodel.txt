1️⃣5️⃣ Technical Architecture:-
Application server:-
1️⃣ Application Server kya hota hai
Application Server ek software platform hota hai jo:-
application code run karta hai
business logic process karta hai
database aur users ke beech connection manage karta hai
Simple definition:-
Application Server = jahan application ka backend code run hota hai

2️⃣ Technical Architecture me role:-
Technical Architecture me flow kuch aisa hota hai:-
User / Browser
      ↓
Web Server
      ↓
Application Server
      ↓
Database Server
Explanation

1️⃣ User:-
User browser se request bhejta hai.

2️⃣ Web Server:-
Request receive karta hai.

3️⃣ Application Server:-
Java code run karta hai
business logic execute karta hai
authentication / authorization check karta hai

4️⃣ Database Server:-
Data store karta hai.

5️⃣ Popular Application Servers:-
Kuch famous application servers:-
Apache Tomcat
JBoss EAP
Oracle WebLogic Server
IBM WebSphere Application Server
IAM tools jaise SailPoint usually Tomcat / WebLogic / JBoss par run karte hain.

4️⃣ IAM / SailPoint me Application Server:-
IAM system me application server ye kaam karta hai:-
Identity processing
Access request workflow
Policy evaluation
Provisioning
Certification

()lekin java code ke sath in xml objects ka bhi use hota hai?
--Haan bhai 👍 Java code ke saath XML objects ka use hota hai.
IAM tools jaise SailPoint IdentityIQ me system XML + Java dono ka combination use karta hai.
Mai simple flow me samjhata hoon.

1️⃣ XML + Java ka relation.
XML → configuration define karta hai
Java → logic run karta hai
Matlab:-
XML me object define hota hai
Java code us object ko read karke execute karta hai

2️⃣ Example 1 – Workflow:-
XML me Workflow
<Workflow name="AccessApproval">
    <Step name="ManagerApproval"/>
</Workflow>
Java code
Workflow wf = context.getObjectByName(Workflow.class, "AccessApproval");
wf.start();
Flow:-
XML → workflow define
Java → workflow run

3️⃣ Example 2 – Application
XML Configuration
<Application name="ActiveDirectory">
    <Connector>LDAPConnector</Connector>
</Application>
Java Code
Application app = context.getObjectByName(Application.class, "ActiveDirectory");
System.out.println(app.getName());

Output:-
ActiveDirectory

4️⃣ Example 3 – Rule:-
Rule bhi XML me hota hai aur uske andar Java code hota hai.

<Rule name="CheckRisk">
    <Source>
        if(identity.getDepartment().equals("Finance")){
            return true;
        }
    </Source>
</Rule>
Flow:-
XML → rule object
Java → rule logic

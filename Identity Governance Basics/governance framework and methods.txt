🏢 1️⃣4️⃣ Governance Framework:-
Governance Framework ek structure hai jo ye decide karta hai ki organization me 
kaun kya karega, kaise rules follow honge, aur responsibilities kaun lega. 
Ye basically ek guideline system hai jo risk ko control karta hai aur 
ensure karta hai ki sab kaam compliant aur efficient tarike se ho.

1️⃣ Purpose (Maksad):-
Governance framework ka main goal hota hai control, compliance aur accountability maintain karna.
Ye ensure karta hai ki policies, laws, regulations aur best practices follow ho rahe hain.

2️⃣ Components (Hissa):-
Ek governance framework me aam taur par ye cheezein hoti hain:
Policies & Standards – Organization ke rules aur standards jo follow karne hai.
Roles & Responsibilities – Kaun decision lega, kaun implement karega.
Processes & Procedures – Kaam ko step by step kaise karna hai.
Monitoring & Reporting – Kaam sahi ho raha hai ya nahi, iska track rakhna.
Risk & Compliance Management – Risks aur laws ka dhyan rakhna.

3️⃣ Example (SailPoint ya IT Governance me):-
Identity Governance me framework define karta hai:
Kaun users ko access de sakta hai
Kaun accounts ko review karega
Kaun approvals lega
Password aur privileged account policies kaise enforce hongi.

()1️⃣ Access Request Lifecycle:-
Ye ek process framework hai jo define karta hai ki user ka access request kaise create, approve aur provision hota hai.
Steps usually:-
Step	                                 Description	                                                                          Example
Request Submission:--------User ya system access request submit karta hai:--------------------------------Employee ko Salesforce ka access chahiye, request form fill karega
Validation:----------------System ya IAM connector check karta hai ki request valid hai ya nahi:----------Check ki user already access to nahi rakhta
Approval:------------------Defined approvers (Approval Matrix) access approve ya reject karte hain:-------Manager approve kare, aur agar privileged access hai to Security Officer bhi approve kare
Provisioning:--------------Access system me allocate hota hai:--------------------------------------------Salesforce account create + role assign
Notification:--------------User ko access grant hone ki info send hoti hai:-------------------------------Email: “Your access is approved”
Review / Recertification:--Periodically check ki access valid hai ya nahi:--------------------------------Quarterly review by manager
Focus:-
Efficient process
Compliance with policies
Audit trail maintain karna

()2️⃣ Approval Matrix:-
Ye decision-making framework hai jo define karta hai ki kaun approvals de sakta hai,
kis level ka access aur kitne approvers chahiye.
Key Points:-
Segregation of Duties (SoD): High-risk access multiple approvals me divide hota hai
Role-based Approval: Low-risk approvals team lead se, high-risk approvals manager + security officer se
Hierarchy / Escalation: Agar primary approver available nahi, to automatic escalate ho

Example Table:-
Access Type	       Approver 1	Approver 2	                       Notes
Standard App:------Team Lead:-------------------------------------Low-risk
HR/Finance App:----Manager:-----Security Officer:-----------------High-risk
Admin/Privileged:--Manager:-----Security Officer + IAM Admin:----Extra control

Focus:-
Accountability ensure karna
Risk reduction via multi-layer approvals

()3️⃣ Control Framework:-
Ye risk & control framework hai jo ensure karta hai ki 
organization ke policies aur regulations follow ho rahe hain.
Components:-
Preventive Controls: Access wrong jagah na mile (password policy, SoD)
Detective Controls: Access misuse detect karna (audit logs, anomaly detection)
Corrective Controls: Issue fix karna (access revoke, alert generation)

Example Controls in IAM:-
Password Complexity & Rotation
Privileged Account Monitoring
Access Review / Recertification
Policy Violations Alerts

Focus:-
Risk management
Operational efficiency
Compliance maintenance

ComplianceReport with generate() – Full Example
import java.util.ArrayList;
import java.util.List;

// Simple PolicyViolation class
class PolicyViolation {

    private String description;

    public PolicyViolation(String description) {
        this.description = description;
    }

    public String getDescription() {
        return description;
    }
}


// Simple Identity class
class Identity {

    private String name;
    private List<String> roles;

    public Identity(String name, List<String> roles) {
        this.name = name;
        this.roles = roles;
    }

    public String getName() {
        return name;
    }

    public List<String> getRoles() {
        return roles;
    }
}


// ComplianceReport class
class ComplianceReport {

    private List<Identity> identities;
    private List<PolicyViolation> violations;

    public ComplianceReport(List<Identity> identities) {
        this.identities = identities;
        this.violations = new ArrayList<PolicyViolation>();
    }

    // 🔹 generate() method
    public void generate() {

        System.out.println("Starting Compliance Report Generation...");

        for (Identity identity : identities) {

            List<String> roles = identity.getRoles();

            // Example SoD Policy
            if (roles.contains("Payment Creator") && roles.contains("Payment Approver")) {

                PolicyViolation violation = new PolicyViolation(
                        identity.getName() + " has SoD violation: Payment Creator + Payment Approver");

                violations.add(violation);
            }
        }

        System.out.println("Compliance Report Generation Completed.");
    }

    public List<PolicyViolation> getViolations() {
        return violations;
    }
}


// Main class
public class Main {

    public static void main(String[] args) {

        List<String> roles1 = new ArrayList<String>();
        roles1.add("Payment Creator");
        roles1.add("Payment Approver");

        List<String> roles2 = new ArrayList<String>();
        roles2.add("Employee");

        Identity user1 = new Identity("John", roles1);
        Identity user2 = new Identity("David", roles2);

        List<Identity> identities = new ArrayList<Identity>();
        identities.add(user1);
        identities.add(user2);

        ComplianceReport report = new ComplianceReport(identities);

        // Generate report
        report.generate();

        // Print violations
        List<PolicyViolation> violations = report.getViolations();

        for (PolicyViolation v : violations) {
            System.out.println(v.getDescription());
        }
    }
}

Compliance Report Generation Completed.
John has SoD violation: Payment Creator + Payment Approver

Specific log search karta hai.

3️⃣ Simple IAM Example:-
public class AuditLogManager {

    public void logEvent(String event) {
        System.out.println("Audit Log: " + event);
    }

    public void getLogs() {
        System.out.println("User created");
        System.out.println("Access approved");
        System.out.println("Role assigned");
    }

    public void queryLogs(String keyword) {

        String[] logs = {
            "User created",
            "Access approved",
            "Finance role assigned"
        };

        for(String log : logs) {

            if(log.contains(keyword)) {
                System.out.println("Matched Log: " + log);
            }

        }
    }

    public static void main(String[] args) {

        AuditLogManager audit = new AuditLogManager();

        audit.logEvent("User requested Finance access");

        audit.getLogs();

        audit.queryLogs("Finance");
    }
}

Output:-
Audit Log: User requested Finance access
User created
Access approved
Role assigned
Matched Log: Finance role assigned

2️⃣ ProvisioningPlan (Real SailPoint Provisioning)
import sailpoint.object.ProvisioningPlan;
import sailpoint.object.ProvisioningPlan.AccountRequest;
import sailpoint.object.ProvisioningPlan.AttributeRequest;

ProvisioningPlan plan = new ProvisioningPlan();
plan.setIdentity(identity);

// account request
AccountRequest accReq = new AccountRequest();
accReq.setApplication("FinanceApp");
accReq.setOperation(AccountRequest.Operation.Create);

// attribute request
AttributeRequest attrReq =
  new AttributeRequest("role",
                       ProvisioningPlan.Operation.Add,
                       "Finance_Read");

accReq.add(attrReq);
plan.add(accReq);

// provisioning execute
Provisioner provisioner = new Provisioner(context);
provisioner.execute(plan);

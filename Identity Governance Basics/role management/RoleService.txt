()Role kya hai?
--"RoleService = Role ka manager + operations ka executor"
RoleService ek service layer ka component hai jo roles ke management ke liye use hota hai.
Yani:-
Ye system me roles create, read, update, delete (CRUD) karta hai.
Ye user aur roles ke beech assignment manage karta hai.
Ye roles ke rules, permissions, aur hierarchy handle karta hai.
💡 Ek tarike se socho:-
"RoleService = system ka role manager" — ye ensure karta hai ki kaunse user ke paas kaunse role hai aur roles properly manage ho rahe hain.

()2️⃣ RoleService kya karta hai?
Ye service mostly ye operations handle karta hai:-
()Role CRUD Operations:-
createRole(Role role) → Naya role create kare.
updateRole(Role role) → Role ke attributes update kare.
deleteRole(String roleId) → Role delete kare.
getRole(String roleId) → Role details fetch kare.
getAllRoles() → System me saare roles fetch kare.

()Role Assignment / Removal:-
assignRole(User user, Role role) → User ko role assign kare.
removeRole(User user, Role role) → User se role remove kare.

()Role Validation / Checking:-
isRoleAssigned(User user, Role role) → Check kare ki user ke paas role hai ya nahi.
validateRole(Role role) → Role ke rules / constraints check kare.

()Role Search / Filtering:-
findRolesByAttribute(String attr, String value) → Attribute ke basis pe roles search kare.
getRolesByUser(User user) → User ke assigned roles fetch kare.

()Role Metadata / Hierarchy:-
getRolePermissions(Role role) → Role ke permissions fetch kare.
getChildRoles(Role role) → Role hierarchy me child roles fetch kare.

3️⃣ RoleService ke andar kya hota hai?
Typically, RoleService ke andar ye cheeze hoti hain:-
()Dependencies / DAOs
()RoleRepository / RoleDAO → Database ya identity engine se roles fetch/update.

()UserService → Users ke sath link handle karne ke liye.

()Core Methods (as explained above):-
--CRUD, Assign/Remove, Search, Validate.

()Business Logic:-
Role ko kisi user pe assign karne se pehle validate karna (eligibility check).
Role deletion se pehle dependencies check karna (e.g., agar role kisi user pe assigned hai toh delete na ho).

()Cache / Optimization (optional):-
RoleService kabhi roles ko memory me cache karke performance improve kar sakta hai.

2️⃣ validateRole(Role role):-
Kya karta hai:-
Ye method role ke rules aur constraints check karta hai, jaise:-
()Name validation:-
Role ka name empty ya null na ho.
Duplicate role name na ho.

()Permissions validation:-
Assigned permissions valid aur existing honi chahiye.

()Hierarchy validation:-
Parent/Child roles me circular dependency na ho.

()Attribute validation:-
Required attributes (jaise department, costCenter) set honi chahiye.

Output:-
Agar sab valid hai → true ya role object wapas aa sakta hai.
Agar invalid hai → Exception throw kar sakta hai (jaise ValidationException)

Example:-
Role role = new Role("Admin");
role.setDepartment("IT");

try {
    roleService.validateRole(role);
    System.out.println("Role valid hai!");
} catch (ValidationException e) {
    System.out.println("Validation failed: " + e.getMessage());
}
// Output: Role valid hai!

import sailpoint.api.SailPointContext;
import sailpoint.api.SailPointContextFactory;
import sailpoint.object.Role;
import sailpoint.tools.GeneralException;
import java.util.List;

public class RoleServiceExample {

    public static void main(String[] args) {

        SailPointContext context = null;

        try {
            // 1️⃣ SailPointContext create karna
            context = SailPointContextFactory.createContext();

            // 2️⃣ Role create karna
            Role newRole = new Role();
            newRole.setName("SuperAdmin");
            newRole.setDescription("This is a SuperAdmin role");

            // Validate role
            try {
                newRole.validate(context);
            } catch (GeneralException ve) {
                System.out.println("Validation failed: " + ve.getMessage());
            }

            // Save role
            context.saveObject(newRole);
            context.commitTransaction();
            System.out.println("Role created: " + newRole.getName());

            // 3️⃣ Role update karna
            Role existingRole = (Role) context.getObjectByName(Role.class, "SuperAdmin");
            existingRole.setDescription("Updated description");
            existingRole.validate(context); // Validate before update
            context.saveObject(existingRole);
            context.commitTransaction();
            System.out.println("Role updated: " + existingRole.getDescription());

            // 4️⃣ Get role by name
            Role fetchedRole = (Role) context.getObjectByName(Role.class, "SuperAdmin");
            System.out.println("Fetched Role: " + fetchedRole.getName());

            // 5️⃣ Get all roles
            List<Role> allRoles = context.getObjects(Role.class);
            System.out.println("All Roles in system:");
            for (int i = 0; i < allRoles.size(); i++) {
                System.out.println(" - " + allRoles.get(i).getName());
            }

            // 6️⃣ Delete role
            Role roleToDelete = (Role) context.getObjectByName(Role.class, "SuperAdmin");
            if (roleToDelete != null) {
                context.deleteObject(roleToDelete);
                context.commitTransaction();
                System.out.println("Role deleted: " + roleToDelete.getName());
            }

        } catch (GeneralException e) {
            e.printStackTrace();
        } finally {
            // 7️⃣ Release context
            if (context != null) {
                SailPointContextFactory.releaseContext(context);
            }
        }
    }
}

()Identity kya karta hai?
Identity = ek user ya account jo system me exist karta hai
Ye SailPoint me central object hai jo kisi employee,contractor,partner ya service account ko represent karta hai.
Simple Example:-
Ravi Kumar → Identity object
Attributes: email, role, department, status, accounts

🔹 2️⃣ Identity ka kaam:-
Kaam	                           Explanation
Access/Entitlements track:-------Identity ke upar assigned roles aur entitlements track karna
Workflow/Policy enforcement:-----Identity changes pe workflows / triggers run karna
Certification:-------------------Certification campaigns me reviewer assign karne me use hota hai
Reporting/Audit:-----------------Audit me identity ke details use hote hain
Event generation:----------------IdentityEvent create hone me base object

🔹 3️⃣ Identity ke andar kya hota hai(Attributes / Fields):-
Field	            Description
id:------------Unique ID of the identity
name:-------------Full name / username
email:---------------Email ID
manager:-------Reporting manager (Identity reference)
status:---------Active / Inactive / Suspended
department:--------------Department name
accounts:------------List of linked system accounts (Account objects)
roles:---------------List of assigned roles
entitlements:---------List of entitlements / permissions
createdDate:---------Identity creation date
modifiedDate:--------Last modification date
certifications:-------Certifications linked to this identity
customAttributes:----Any additional custom attribute (key-value)

🔹5️⃣ DB me Identity ka storage:-
Yes bhai, Identity DB me store hota hai
Tables usually:
Identity → basic fields (id, name, email, status, manager_id, department, createdDate, modifiedDate)
Identity_Accounts → linked accounts
Identity_Roles → assigned roles
Identity_Entitlements → permissions / entitlements
Identity_Certifications → linked certifications
Ye DB me store hone ke baad SailPoint workflow aur triggers ke saath integrate hota hai.

🔹Manager field kya hai?
manager = Identity object ka reporting manager
Matlab ye us bande ka reference hai jo directly identity ka boss hai
Ye Identity reference type ka hota hai, yani ek Identity object ke andar dusra Identity object store hota hai

Example:-
Employee Identity	Manager Identity
Ravi Kumar	Sunil Sharma
Priya Verma	Sunil Sharma
Yaha Ravi Kumar aur Priya Verma dono ki manager field me Sunil Sharma ki Identity reference store hai.

()🔹Ye kaam kya karta hai?
Certification / Approval workflow me:-
Jab koi certification campaign run hota hai, manager ko reviewer assign karna hota hai → manager field use hoti hai.

()Trigger/Event automation:-
Example: Jab employee ka role change ho → manager ko notification / approval send karna.

1️⃣ getId():-
Purpose: Identity ka unique ID return karta hai
identity.getId(); // "ID_1001"

2️⃣ setId(String id):-
Purpose: Identity ka ID set karta hai
identity.setId("ID_1001");

3️⃣ hasId():-
Purpose: Check karta hai ID set hai ya nahi
identity.hasId(); // true

4️⃣ getName():-
Purpose: Identity ka name return karta hai
identity.getName(); // "Ravi Kumar"

5️⃣ setName(String name):-
Purpose: Identity ka name set karta hai
identity.setName("Ravi Kumar");

6️⃣ hasName():-
Purpose: Name set hai ya nahi check karta hai
identity.hasName(); // true

7️⃣ getEmail():-
Purpose: Identity ka email return karta hai
identity.getEmail(); // "ravi.kumar@company.com"

8️⃣ setEmail(String email):-
Purpose: Email set karta hai
identity.setEmail("ravi.kumar@company.com");

9️⃣ hasEmail():-
Purpose: Email exist karta hai ya nahi check karta hai
identity.hasEmail(); // true

🔟 getManager():-

()resolveDisplayName() SailPoint IdentityIQ (IIQ) ka ek bohot hi useful utility method hai, 
jo aamtaor par sailpoint.object.Identity class ya SailPoint ki internal API classes (jaise sailpoint.api.ObjectUtil) ka hissa
hota hai.
resolveDisplayName thik yahi kaam karta hai:-
Aap is method ko koi Identity Name (ID) dete hain.
Yeh database me jaakar us Identity ka displayName dhoodhta hai (jaise: First Name + Last Name).
Agar user ka display name set hai, toh yeh use return kar dega (e.g., "Rahul Sharma").
Backup Plan: Agar kisi user ka display name set nahi hai (null hai), toh yeh workflow ko crash nahi karta, 
balki backup me uska Unique Username/ID hi return kar deta hai taaki screen khali na dikhe.

1️⃣ IdentityTrigger kya hai?
IdentityTrigger = Ye ek object hai jo Identity ke upar automatic actions / events trigger karne ke liye use hota hai.
Basically, IdentityTrigger = “rules / automation jo Identity changes pe run hote hain”
Example:-
Jab koi naya user create ho → automatic account create kar do,
Jab koi role assign ho → notification bhej do.

🔹2️⃣ IdentityTrigger ka kaam:-
()Automation:-
Identity ke changes pe automatically actions run karna (create accounts, assign roles, disable access).
()Event-based triggers:-
IdentityEvent generate hone pe trigger hota hai.
()Workflow / Rule Integration:-
Workflows ya custom rules execute karna.

🔹 3️⃣ IdentityTrigger ke andar kya hota hai?
Field/Attribute	             Description
id:---------------------Unique ID of the trigger
name:-----------------------Trigger ka name
description:-------------Trigger ka description
eventType:----------Kis type ke IdentityEvent pe trigger hoga (Create / Update / RoleAssigned / EntitlementRemoved)
conditions:------------Conditions list jinke satisfy hone par trigger fire hota hai
actions:------------Actions list jo trigger execute karega (assign role, send notification, disable account)
enabled:------------Trigger active hai ya nahi (true / false)
createdDate:--------Kab trigger create hua
modifiedDate:-------Last modified date

🔹 6️⃣ DB me store hota hai?
✅ Haan bhai, IdentityTrigger DB me store hota hai, taaki system restart ke baad bhi triggers active rahe.
Tables me Trigger ID, Name, EventType, Conditions, Actions, Enabled flag store hota hai,
Ye table IdentityEvent aur Identity object ke sath link hota hai.

🔹 1️⃣ Condition ka nature:-
Trigger me condition = event fire hone ke liye jo check lagta hai, matlab filter ya rule.
Ye teen tarah ka ho sakta hai:-

A) Predefined (built-in) conditions:-
SailPoint me kuch common conditions pehle se defined hote hain,
Example:-
status = Active
role = Finance
identityType = Employee
Ye conditions directly dropdown ya property se select kiye ja sakte hain,
Condition c = new Condition("status", "Active"); // Predefined condition
trigger.setConditions(List.of(c));

B) Custom rule-based conditions:-
Ye developer define karta hai using BeanShell / JavaScript / custom rule class
Example:-
Agar user ka department = IT aur location = Mumbai → trigger fire karo
Condition customCondition = new Condition();
customCondition.setRule("return identity.getAttribute('department').equals('IT') && identity.getAttribute('location').equals('Mumbai');");
trigger.setConditions(List.of(customCondition));
Ye runtime pe evaluate hota hai, Flexible,complex logic implement karne ke liye.

C) Normal code / expression-based conditions:-
Kabhi kabhi simple attribute comparison code hi condition hota hai
Example:-
Condition c = new Condition();
c.setExpression("identity.getAttribute('jobTitle').equals('Manager')");
trigger.setConditions(List.of(c));
Ye predefined se zyada flexible hai, par easy maintainable bhi hai.

🔹 1️⃣ Action kya hai?
Action = IdentityTrigger fire hone par perform hone wali operation/command,
Matlab trigger fire → action execute
Examples:
Role assign karna
Entitlement revoke karna
Email send karna
Account disable karna

🔹 2️⃣ Action predefined hai ya custom?
()Predefined Actions (built-in):-
SailPoint me common actions pehle se define hote hain
Example:-
AssignRoleAction → Role assign karta hai
RemoveRoleAction → Role remove karta hai
SendEmailAction → Email send karta hai
DisableAccountAction → Account disable karta hai
Action emailAction = new SendEmailAction("admin@company.com", "Welcome to system!");
trigger.setActions(List.of(emailAction));

()Custom Actions:-
Developer apna Action class bana sakta hai by extending Action class
Example: CustomNotificationAction, CustomWorkflowAction
public class CustomNotificationAction extends Action {
    @Override
    public void execute(Identity identity) {
        System.out.println("Custom notification for " + identity.getName());
    }
}
Action customAction = new CustomNotificationAction();
trigger.setActions(List.of(customAction));

import sailpoint.api.SailPointContext;
import sailpoint.api.SailPointContextFactory;
import sailpoint.object.Identity;
import sailpoint.object.IdentityTrigger;
import sailpoint.object.Condition;
import sailpoint.object.Action;
import sailpoint.object.EmailAction;
import sailpoint.tools.GeneralException;

import java.util.Date;
import java.util.List;

public class IdentityTriggerExample {

    public static void main(String[] args) {
        SailPointContext context = null;

        try {
            // 1️⃣ Get SailPoint Context
            context = SailPointContextFactory.getContext();

            // 2️⃣ Retrieve an Identity
            Identity user = context.getObjectByName(Identity.class, "ravi.kumar");

            // 3️⃣ Create a new IdentityTrigger
            IdentityTrigger trigger = new IdentityTrigger();
            trigger.setId("TRG_1001");
            trigger.setName("New Identity Welcome Trigger");
            trigger.setDescription("Send welcome email to new identities");
            trigger.setEventType("Create");
            trigger.setEnabled(true);
            trigger.setCreatedDate(new Date());
            trigger.setModifiedDate(new Date());

            // 4️⃣ Set Conditions (List<Condition>)
            Condition activeStatusCondition = new Condition("status", "Active"); // Predefined condition
            Condition deptCondition = new Condition(); // Custom rule condition
            deptCondition.setRule("return identity.getAttribute('department').equals('IT');");
            trigger.setConditions(List.of(activeStatusCondition, deptCondition));

            // 5️⃣ Set Actions (List<Action>)
            Action welcomeEmailAction = new EmailAction("admin@company.com", 
                "Welcome to the system!", "Hello " + user.getName());
            // You can also add custom action here if needed
            trigger.setActions(List.of(welcomeEmailAction));

            // 6️⃣ Save the trigger in context (DB)
            context.saveObject(trigger);
            context.commitTransaction();

            // 7️⃣ Retrieve and print the trigger
            IdentityTrigger retrievedTrigger = context.getObjectById(IdentityTrigger.class, "TRG_1001");
            System.out.println("Retrieved Trigger: " + retrievedTrigger.toString());

        } catch (GeneralException e) {
            e.printStackTrace();
        } finally {
            if (context != null) {
                SailPointContextFactory.closeContext(context);
            }
        }
    }
}

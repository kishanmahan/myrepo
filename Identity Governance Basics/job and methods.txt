🔹 2️⃣ Job:-
Kya hai:-
Job ek bada unit of work ya scheduled execution hota hai.
Ye ek ya multiple tasks ka collection hota hai jo ek saath execute hote hain.
Mostly background / periodic process ke liye hota hai.

Example:-
IdentityRefreshJob → saare identities ko daily refresh kare.
AccessCertificationJob → access review workflow trigger kare for multiple users.

Key points:-
Job usually scheduled / automated hota hai (daily, weekly, etc.).
Job me multiple tasks ho sakte hain.
Long-running aur system-wide impact wale processes ke liye use hota hai.

import sailpoint.api.SailPointContext;
import sailpoint.object.Job;
import sailpoint.object.Identity;
import sailpoint.api.JobManager;
import java.util.Date;

public class JobExample {

    public static void main(String[] args) {

        SailPointContext context = null;

        try {
            // Get SailPoint Context (runtime)
            context = SailPointFactory.createContext();

            // Fetch Job by name or id
            String jobName = "My Job Name";
            Job myJob = context.getObject(Job.class, jobName);

            if (myJob == null) {
                System.out.println("Job not found: " + jobName);
                return;
            }

            // ----------------------------
            // 1️⃣ Identification / Basic Info
            // ----------------------------

            System.out.println("Job ID: " + myJob.getId());
            System.out.println("Job Name: " + myJob.getName());
            System.out.println("Job Type: " + myJob.getType());

            // Owner
            Identity owner = myJob.getOwner();
            if (owner != null) {
                System.out.println("Job Owner: " + owner.getName());
            }

            // Set Owner
            Identity newOwner = context.getObjectByName(Identity.class, "admin");
            myJob.setOwner(newOwner);
            context.commitTransaction();

            // ----------------------------
            // 2️⃣ Status / Lifecycle
            // ----------------------------

            System.out.println("Current Status: " + myJob.getStatus());

            // Launch immediately
            JobManager.launch(myJob, context);
            System.out.println("Launched Job.");

            // Set status manually
            myJob.setStatus(Job.Status.Running.name());
            System.out.println("Updated Status: " + myJob.getStatus());

            // Cancel job
            JobManager.cancel(myJob, context);
            System.out.println("Canceled Job.");

            // Complete job
            myJob.setStatus(Job.Status.Completed.name());
            System.out.println("Completed Job.");

            // Reject job
            myJob.setStatus(Job.Status.Rejected.name());
            System.out.println("Rejected Job.");

            // Archive job
            myJob.archive(context);
            System.out.println("Archived Job.");

            context.commitTransaction();

            // ----------------------------
            // 3️⃣ Audit / Metadata
            // ----------------------------

            System.out.println("Job Created: " + myJob.getCreated());
            System.out.println("Job Modified: " + myJob.getModified());

            if (myJob.getCreatedBy() != null)
                System.out.println("Created By: " + myJob.getCreatedBy().getName());

            if (myJob.getModifiedBy() != null)
                System.out.println("Modified By: " + myJob.getModifiedBy().getName());

        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            if (context != null) {
                context.decache(true);
                context.close();
            }
        }
    }
}

()validate() kya-kya check karta hai:-

1️⃣ Required fields check:-
Agar koi mandatory field missing hai (jaise job name, owner, arguments, target application), to exception throw karta hai.

job.setArguments(null); 
job.validate(); // → IllegalArgumentException ya ValidationException

2️⃣ Argument types / values:-
Check karta hai ki jo arguments diye gaye hain, unka type aur value sahi hai ya nahi.
Example: String expected hai, lekin number pass ho gaya → error.

3️⃣ Status consistency:-
Job ka status valid hona chahiye. Jaise RUNNING, COMPLETED, FAILED.
Agar invalid status set hai → validation fail.

4️⃣ Execution constraints:-
Agar koi time constraints ya scheduling rules defined hain, to ye check karta hai ki execution valid hai.

5️⃣ Workflow / Task references:-
Agar job kisi workflow ya task se linked hai, to validate karta hai ki linked object exist karta hai aur accessible hai.

6️⃣ Custom rules:-
Developer ne agar koi custom validation rules define kiye hain, to validate() unhe bhi check karega.
Example:-
job.setArguments(Map.of()); // empty arguments
job.setExecutionStatus("RUNNING"); 
job.validate(); // agar arguments required hain → ValidationException

Output:-
ValidationException: Arguments cannot be empty

()Job TargetType (Target / Integration):-
Job ka TargetType basically ye batata hai ki job kis tarah ke system, object, ya resource par kaam karega.
Ye mostly workflow, provisioning, integration ke liye use hota hai.
Common / Standard TargetTypes in SailPoint:-

1️⃣ Application:-
Job kisi connected application (Active Directory, HR System, ERP, SailPoint Application) ke liye run ho raha hai.

2️⃣ System:-
Job kisi external system ya server ke liye run ho raha hai (non-application, generic system).

3️⃣ Identity:-
Job directly Identity object (user) par kaam kar raha hai, jaise create, update, disable user.

4️⃣ Task:-
Job kisi specific Task ko execute kar raha hai.

5️⃣ Workflow:-
Job kisi WorkflowCase ya workflow process ko execute kar raha hai.

6️⃣ Role:-
Job role assignments ya role-based provisioning ke liye.

7️⃣ Group:-
Job kisi user group ke liye actions perform kar raha hai.

8️⃣ Policy:-
Job kisi policy evaluation ya policy-based action ke liye.

9️⃣ Certification:-
Job kisi access review / certification campaign ke liye run ho raha hai.

🔟 Custom / Other:-
Agar koi custom integration ya unique object target hai, to type Custom ya user-defined value ho sakta hai.

import sailpoint.api.SailPointContext;
import sailpoint.api.JobManager;
import sailpoint.object.Job;
import sailpoint.object.Task;
import sailpoint.object.WorkItem;
import sailpoint.object.ApprovalSet;
import sailpoint.object.Identity;

import java.util.List;
import java.util.Map;

public class JobExample {

    public static void main(String[] args) {

        SailPointContext context = null;
        try {
            // 🔹 Get SailPoint context
            context = SailPointFactory.createContext();

            // 🔹 Load Job by name or ID
            String jobName = "Quarterly Certification Job";
            Job job = context.getObject(Job.class, jobName);

            if (job == null) {
                System.out.println("Job not found: " + jobName);
                return;
            }

            // ============================================
            // 1️⃣ Basic Info
            // ============================================
            System.out.println("Job ID: " + job.getId());
            System.out.println("Job Name: " + job.getName());
            System.out.println("Job Type: " + job.getType());

            Identity owner = job.getOwner();
            System.out.println("Job Owner: " + (owner != null ? owner.getName() : "None"));

            // ============================================
            // 2️⃣ Tasks Related Methods
            // ============================================
            List<Task> allTasks = job.getTasks();
            System.out.println("All Tasks in Job: " + allTasks.size());

            List<Task> pendingTasks = job.getPendingTasks();
            System.out.println("Pending Tasks: " + pendingTasks.size());

            List<Task> completedTasks = job.getCompletedTasks();
            System.out.println("Completed Tasks: " + completedTasks.size());

            // Get specific Task by ID
            Task t1 = job.getTaskById("TASK123");
            System.out.println("Task TASK123: " + t1);

            // Get Task Results
            Map<String, Object> results = job.getTaskResults();
            System.out.println("Task Results: " + results);

            // ============================================
            // 3️⃣ WorkItems Related Methods
            // ============================================
            List<WorkItem> workItems = job.getWorkItems();
            System.out.println("Total WorkItems in Job: " + workItems.size());

            List<WorkItem> pendingWorkItems = job.getPendingWorkItems();
            System.out.println("Pending WorkItems: " + pendingWorkItems.size());

            List<WorkItem> completedWorkItems = job.getCompletedWorkItems();
            System.out.println("Completed WorkItems: " + completedWorkItems.size());

            WorkItem wi = job.getWorkItemById("WI789");
            System.out.println("WorkItem WI789: " + wi);

            // ============================================
            // 4️⃣ Approval Set
            // ============================================
            ApprovalSet approvalSet = job.getApprovalSet();
            System.out.println("Approval Set: " + approvalSet);

            // Optionally update approval set
            // job.setApprovalSet(newApprovalSet);

            // ============================================
            // 5️⃣ Target Info
            // ============================================
            Object target = job.getTarget();
            System.out.println("Job Target: " + (target != null ? target.toString() : "None"));
            System.out.println("Target Name: " + job.getTargetName());
            System.out.println("Target Type: " + job.getTargetType());

            context.commitTransaction();

        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            if (context != null) {
                context.decache(true);
                context.close();
            }
        }
    }

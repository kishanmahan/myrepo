🔥 1. BeanShell kya hai?
BeanShell ek scripting language hai (Java jaisa syntax)
Iska use hota hai code likhne ke liye
👉 Matlab:-
“Kaise kaam hoga” ye BeanShell batata hai
Example:-
String name = "Kishan";
System.out.println(name);

🔥 2. Rule kya hai?
Rule ek SailPoint ka object hai
Ye system me store hota hai (DB me)
Iske andar BeanShell code likha hota hai
👉 Matlab:-
“Kab aur kahan kaam hoga” ye Rule decide karta hai.
----------------------------------------------------------
🔥 3. Relation (Sabse important 💥):-
👉 Simple line:-
Rule = Container + BeanShell Code
Cheez	        Meaning
BeanShell	   Code (logic)
Rule	       Wrapper/Container (jisme code likha hota hai)
--------------------------------------------------------------
🔥 4. Real Example (SailPoint):-
<Rule name="ExampleRule" type="BeforeProvisioning">
    <Source>
        String username = identity.getName();
        System.out.println("User: " + username);
    </Source>
</Rule>
👉 Yaha:-
<Rule> = Rule object
<Source> ke andar = BeanShell code

🧠 Rule kya karta hai?
👉 Rule use hota hai:-

🔹 1️⃣ Custom Logic likhne ke liye:-
Data transform karna
Conditions lagana

🔹 2️⃣ Integration ke liye:-
External system se data lena
Modify karna

🔹 3️⃣ Provisioning control:-
Account create/update/delete ka logic

🔹 4️⃣ Validation:-
Data sahi hai ya nahi check karna

🔹 5️⃣ Automation:-
Auto decisions lena

📦 Rule ke andar kya hota hai?
👉 Ek rule me mainly ye hota hai:-
🔹 Structure:-
<Rule name="Sample Rule" type="IdentityAttribute">
    <Source>
        // Beanshell / Java code
        return "Hello";
    </Source>
</Rule>
🔹 Important Parts:-
Name → Rule ka naam
Type → Rule kis kaam ke liye hai
Source → Actual code (logic)

🔥 Real Production BeforeProvisioning Rule:-
👉 Use-case:
Jab user ka access kisi critical system (jaise AD) me ja raha ho, to check karo ki user valid hai aur mandatory attribute present hai.
Agar nahi hai → provisioning block karo ❌
🔥 Ye Rule Live me kya karega?
✔ Provisioning start hone se pehle run hoga
✔ Identity ka data check karega
✔ Agar required data missing → Exception throw → provisioning fail ❌
✔ Agar valid → provisioning continue ✅

<Rule name="BP_ValidateUser_NoStarImport" type="BeforeProvisioning">
    <Source>

        import sailpoint.object.Identity;
        import sailpoint.object.ProvisioningPlan;
        import sailpoint.tools.GeneralException;

        // Get Identity from provisioning plan
        Identity identity = provisioningPlan.getIdentity();

        if(identity == null) {
            throw new GeneralException("Identity not found in provisioning plan");
        }

        // Get attributes
        String empId = (String) identity.getAttribute("employeeId");
        String dept  = (String) identity.getAttribute("department");

        // Validation
        if(empId == null || empId.trim().equals("")) {
            throw new GeneralException("Provisioning blocked: employeeId is missing");
        }

        if(dept == null) {
            throw new GeneralException("Provisioning blocked: department is null");
        }

        // Business logic
        if("IT".equalsIgnoreCase(dept)) {
            // allow provisioning
        } else {
            throw new GeneralException("Provisioning blocked: only IT users allowed");
        }

    </Source>
</Rule>

<Rule name="BP_Enterprise_WithContext" type="BeforeProvisioning">
    <Source>

        // Explicit imports (NO *)
        import sailpoint.api.SailPointContext;
        import sailpoint.object.Identity;
        import sailpoint.object.ProvisioningPlan;
        import sailpoint.object.AccountRequest;
        import sailpoint.object.AttributeRequest;
        import sailpoint.tools.GeneralException;

        import java.util.List;

        // Context (NOTE: Proper way in many environments)
        SailPointContext context = context;

        // ProvisioningPlan (IMPORTANT: ye runtime me automatically available hota hai)
        ProvisioningPlan plan = provisioningPlan;

        if(plan == null) {
            throw new GeneralException("ProvisioningPlan is null");
        }

        // Identity from plan
        Identity identity = plan.getIdentity();

        if(identity == null) {
            throw new GeneralException("Identity is null in provisioning plan");
        }

        // Attributes
        String empId = (String) identity.getAttribute("employeeId");
        String dept  = (String) identity.getAttribute("department");

        // Validation
        if(empId == null || empId.trim().isEmpty()) {
            throw new GeneralException("Provisioning blocked: employeeId missing");
        }

        if(dept == null || dept.trim().isEmpty()) {
            throw new GeneralException("Provisioning blocked: department missing");
        }

        // Fetch latest identity from DB
        Identity dbIdentity = context.getObjectByName(Identity.class, identity.getName());

        if(dbIdentity == null) {
            throw new GeneralException("Identity not found in DB");
        }

        String managerName = (String) dbIdentity.getAttribute("manager");

        if(managerName == null || managerName.trim().isEmpty()) {
            throw new GeneralException("Provisioning blocked: manager not assigned");
        }

        // Business rule
        if(!"IT".equalsIgnoreCase(dept)) {
            throw new GeneralException("Provisioning blocked: only IT allowed");
        }

        // Loop AccountRequests
        List<AccountRequest> accountRequests = plan.getAccountRequests();

        if(accountRequests != null) {

            for(AccountRequest accReq : accountRequests) {

                String op = accReq.getOperation().toString();

                if(!"Create".equalsIgnoreCase(op) && !"Modify".equalsIgnoreCase(op)) {
                    throw new GeneralException("Unsupported operation: " + op);
                }

                // Add custom attribute
                AttributeRequest attr = new AttributeRequest();
                attr.setName("source");
                attr.setValue("BeforeProvisioningRule");
                attr.setOperation(ProvisioningPlan.Operation.Add);

                accReq.add(attr);
            }
        }

        // Logging
        System.out.println("BeforeProvisioning executed for: " + empId);

    </Source>
</Rule>

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE Rule PUBLIC "sailpoint.dtd" "sailpoint.dtd">
<Rule language="beanshell" name="Global-BeforeProvisioning-Rule" type="BeforeProvisioning">
  <Description>
    Production Rule: 
    1. Logs every request for auditing.
    2. Enforces lowercase on sAMAccountName.
    3. Handles dynamic OU placement for Active Directory.
  </Description>
  <Source>
  <![CDATA[
  import sailpoint.object.*;
  import sailpoint.api.ProvisioningPlan;
  import sailpoint.api.ProvisioningPlan.AccountRequest;
  import sailpoint.api.ProvisioningPlan.AttributeRequest;
  import org.apache.commons.logging.Log;
  import org.apache.commons.logging.LogFactory;

  // Live environment mein logs dekhne ke liye logger initialize karein
  Log customLogger = LogFactory.getLog("com.company.identity.rule.BeforeProvisioning");

  if (plan != null) {
      customLogger.info("Starting BeforeProvisioning Rule for Plan: " + plan.getNativeId());

      List accountRequests = plan.getAccountRequests();
      if (accountRequests != null) {
          for (AccountRequest accReq : accountRequests) {
              String appName = accReq.getApplicationName();
              AccountRequest.Operation op = accReq.getOperation();
              
              customLogger.info("Processing " + op + " request for Application: " + appName);

              // Sirf Active Directory aur Create operation ke liye logic
              if (appName.contains("Active Directory") && AccountRequest.Operation.Create.equals(op)) {
                  
                  // 1. Lowercase Enforcement (Security & Standards)
                  AttributeRequest sam = accReq.getAttributeRequest("sAMAccountName");
                  if (sam != null && sam.getValue() != null) {
                      String originalValue = (String) sam.getValue();
                      sam.setValue(originalValue.toLowerCase());
                      customLogger.debug("Normalized sAMAccountName to lowercase: " + sam.getValue());
                  }

                  // 2. Department-based OU Placement
                  // Identity object se attribute nikalna
                  String dept = "Unknown";
                  Identity iden = plan.getIdentity();
                  if(iden != null) {
                      dept = (String) iden.getAttribute("department");
                  }

                  String parentDN = "OU=Standard,DC=production,DC=com"; // Default OU
                  if ("Finance".equalsIgnoreCase(dept)) {
                      parentDN = "OU=Finance,OU=Accounts,DC=production,DC=com";
                  } else if ("IT".equalsIgnoreCase(dept)) {
                      parentDN = "OU=IT,OU=Users,DC=production,DC=com";
                  }

                  // Provisioning Plan mein dynamically OU add karna
                  accReq.add(new AttributeRequest("AC_NewParent", ProvisioningPlan.Operation.Set, parentDN));
                  customLogger.info("Assigned OU: " + parentDN + " for Department: " + dept);
              }
          }
      }
  }
  customLogger.info("BeforeProvisioning Rule Execution Completed.");
  ]]>
  </Source>
</Rule>

()✔️ Kaise kaam karta hai:-
Ye Apache Commons Logging ka logger hai
String me jo naam diya (com.company.identity.rule.BeforeProvisioning) → wahi logger ka category name ban jata hai
Usually logging frameworks (Log4j / Logback) ke saath internally connect hota hai.

()Log logger = LogFactory.getLog(this.getClass());:-
🔹 this.getClass() kya karta hai?
Log logger = LogFactory.getLog(this.getClass());
✔️ Meaning:-
this → current object (jis class me code likha hai)
getClass() → us object ki runtime class return karta hai
👉 Iska matlab:-
"jis class me ye code chal raha hai, usi class ke naam se logger banega"

🔥 Example samajh:-
Class:-
public class MyRule {
    public void execute() {
        Log logger = LogFactory.getLog(this.getClass());
        logger.info("Hello");
    }
}
Internally kya hoga?
this.getClass() → MyRule.class
Logger ka naam banega:-
com.company.MyRule

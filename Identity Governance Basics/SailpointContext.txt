1️⃣ commitTransaction():-
🔹 Description:-
commitTransaction() database me jo bhi changes kiye gaye hain unko permanently save (commit) karta hai.
Jab bhi tu:-
user update karta hai
role assign karta hai
entitlement change karta hai
provisioning execute karta hai
tab tak changes temporary transaction state me rehte hain.
commitTransaction() run hone par hi database me save hote hain.

2️⃣ rollbackTransaction():-
🔹 Description:-
Agar transaction me error aa jaye ya changes cancel karne ho, tab rollbackTransaction() use hota hai.
Ye transaction ke saare changes undo kar deta hai.
Example code:-
try{
Identity user = context.getObjectByName(Identity.class,"rahul");
user.setDisplayName("Rahul Kumar");
context.saveObject(user);
context.commitTransaction();
}catch(Exception e){
context.rollbackTransaction();
}

3️⃣ context.provision(project)
🔹 Description:-
Ye method ProvisioningProject execute karta hai.
ProvisioningProject me jo instructions hoti hain:
account create
account modify
entitlement add
role assign
wo target application me run hoti hain.

🔹 Example:-
ProvisioningProject project =
        new ProvisioningProject(plan);

context.provision(project);

context.commitTransaction();

🔹 Flow:-
ProvisioningPlan
       ↓
ProvisioningProject
       ↓
context.provision(project)
       ↓
Connector Run
       ↓
Target System Update

4️⃣ context.refreshObject(user):-
🔹 Description:-
Ye method object ko database se dobara reload / refresh karta hai.
Kab use hota hai:-
agar object outdated ho
agar multiple processes update kar rahe ho
agar latest data chahiye

🔹 Example:-
Identity user =
        context.getObjectByName(Identity.class,"rahul");

context.refreshObject(user);

System.out.println(user.getDisplayName());

5️⃣ search(Class type, QueryOptions qo, String fields)
🔹 Description:-
Ye method database me search run karta hai filter ke saath.
Aur sirf specific fields return karta hai (performance optimization ke liye).

🔹 Parameters:-
Parameter	        Meaning
type:---------kaunsa object search karna hai
qo:-----------query filter conditions
fields:-------kaun se fields return karne hain

🔹 Example:-
QueryOptions qo = new QueryOptions();

qo.addFilter(Filter.eq("department","IT"));

Iterator it = context.search(
        Identity.class,
        qo,
        "name"
);
-------------------------------------------------------------------------------
💻 Example:-
import sailpoint.object.Identity;
import sailpoint.object.QueryOptions;
import sailpoint.object.Filter;

QueryOptions qo = new QueryOptions();
qo.addFilter(Filter.eq("inactive", false));

int count = context.countObjects(Identity.class, qo);

log.info("Total Active Identities: " + count);
🔥 Output:-
Total Active Identities: 150
------------------------------------
💻 Example:-
import sailpoint.object.Identity;
import sailpoint.object.QueryOptions;
import sailpoint.object.Filter;

QueryOptions qo = new QueryOptions();
qo.addFilter(Filter.eq("name", "raj"));

Identity identity = (Identity) context.getUniqueObject(Identity.class, qo);

if (identity != null) {
    log.info("Identity Found: " + identity.getName());
}
🔥 Output:-
Identity Found: raj

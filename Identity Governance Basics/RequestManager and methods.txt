1️⃣ RequestManager kya hai?
RequestManager ek service/utility class hai jo Request objects aur workflows ke operations ko handle karta hai.
Ye basically RequestContext aur RequestDefinition ke upar kaam karta hai.
Iska kaam hai request ko system me execute karna, approvals manage karna, workflow cases create karna, aur runtime state handle karna.

2️⃣ RequestManager ka kaam / Responsibilities:-
()Request Create karna:-
RequestDefinition ke template ke basis par nayi Request object create karta hai.
Example: Access request, Service request, Certification request.

()Request Execute karna:-
Request ko workflow engine ke through execute karna.
Approvers ko assign karna aur approvals track karna.

()Approval Management:-
Approval items add/remove/update karna.
Approver ke decisions (approve/reject) process karna.

()Validation Rules Apply Karna:-
Request submit hone se pehle validation rules check karta hai.

()RequestContext Generate Karna:-
Execution ke dauran RequestContext create karta hai aur runtime info store karta hai.

()WorkflowCase Integration:-
Request ke liye workflowcase create karna aur uske steps track karna.

()Comments & Notifications:-
Approver comments capture karna
Notification templates ke through alerts ya emails send karna

()Persistence / DB Operations:-
Requests aur approvals ka data database me save karna.

()1️⃣ RequestContext create karna:-
Method: RequestManager.getRequestContext(Request request)
Purpose:-
Ye method nayi RequestContext object generate karta hai runtime me kisi specific Request ke liye.
RequestContext me automatically RequestDefinition, Attributes, ApprovalItems, Current WorkItem, WorkflowCase attach ho jaate hain.

Example:-
Request request = requestManager.getRequestById("REQ123");

// Runtime context create
RequestContext context = requestManager.getRequestContext(request);

System.out.println(context.getRequest().getId());      // REQ123
System.out.println(context.getState());               // Pending (default)

Output:-
REQ123
Pending

Note:-
Ye method DB se Request fetch karke runtime context ready karta hai.
Agar workflow execution start karni ho, ye mandatory hota hai.

2️⃣ WorkflowCase create karna:-
Method: WorkflowEngine.createWorkflowCase(RequestContext context)
SailPoint ke internal workflow engine ke methods me hota hai.
Purpose:-
Ye method RequestContext ke liye workflow case create karta hai, jisme workflow steps, approvals aur notifications track hote hain.
Example:-
WorkflowCase wfCase = workflowEngine.createWorkflowCase(context);
context.setWorkflowCase(wfCase);
System.out.println(context.getWorkflowCase().getId());
Output (example):-
WFCASE-456

RequestContext aur WorkflowCase ek tight pair hai — RequestContext me WorkflowCase set karna mandatory hai,
taaki approvals aur workflow steps track ho sake.

1️⃣ createRequest(RequestDefinition def):-
Purpose:-
Ye method RequestDefinition (template) ke base par naya Request object create karta hai.
Ye sirf object banata hai, abhi execute nahi karta.

Example:-

def.setName("AccessRequest");
Request request = requestManager.createRequest(def);

Output:-

2️⃣ submitRequest(Request request):-
Purpose:-
Ye method request ko submit karta hai aur workflow start karta hai.
Yahi se actual execution start hota hai (approval, validation, etc).

Example:-
requestManager.submitRequest(request);
System.out.println("Request submitted");
Output:-

🔥 1️⃣ executeRequest(RequestContext context):-
📌 Purpose:-
Runtime me request ko actual execute karta hai
Workflow steps, approvals, validations sab yahin run hote hain
⚙️ Internally:-
RequestContext load hota hai
WorkflowCase attach hota hai
Current WorkItem create hota hai
Approval flow start hota hai
State update hota hai (Pending → InProgress → Completed)
🧪 Example:-
RequestContext context = requestManager.getRequestContext(request);
requestManager.executeRequest(context);

System.out.println(context.getState());
✅ Output:-
Pending / Completed

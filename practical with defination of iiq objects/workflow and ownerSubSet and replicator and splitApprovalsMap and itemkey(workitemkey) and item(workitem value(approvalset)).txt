Aasan bhasha mein kahein toh: Jab approvalSplitPoint ek bade provisioning plan ko alag-alag chote plans (sub-plans) mein tod deta

<!-- 1. Pehle hum Master ApprovalSet ko tod kar Split Approval Map banate hain -->
<Step name="Build Split Approval Set" resultVariable="splitApprovalsMap">
  <Description>
    Yeh step master approval set ko read karta hai aur approver-wise alag-alag ApprovalSet objects ka Map taiyar karta hai.
  </Description>
  <Script>
    <Source>
      <![CDATA[
        import sailpoint.object.ApprovalSet;
        import sailpoint.object.ApprovalItem;
        import java.util.Map;
        import java.util.HashMap;

        ApprovalSet masterSet = (ApprovalSet) workflow.get("approvalSet");
        Map splitMap = new HashMap();

        if (masterSet != null && !masterSet.isEmpty()) {
            for (ApprovalItem item : masterSet.getItems()) {
                // Maan lijiye humne har application ka ek dedicated owner define kiya hua hai
                String ownerName = "spadmin"; 
                if ("Active Directory".equals(item.getApplication())) {
                    ownerName = "adManager";
                } else if ("SAP".equals(item.getApplication())) {
                    ownerName = "sapOwner";
                }

                // Agar is owner ka pehle se koi bucket nahi bana toh naya ApprovalSet banao
                ApprovalSet subSet = (ApprovalSet) splitMap.get(ownerName);
                if (subSet == null) {
                    subSet = new ApprovalSet();
                    splitMap.put(ownerName, subSet);
                }
                
                // Item ko uske sahi owner ke bucket me daal do
                subSet.add(new ApprovalItem(item));
            }
        }
        return splitMap;
      ]]>
    </Source>
  </Script>
  <Transition to="Generate Parallel Approvals"/>
</Step>

<!-- 2. Ab is tayar splitMap ka use karke hum parallel approvals trigger karte hain -->
<Step name="Generate Parallel Approvals" lineItem="ownerSubSet" replicator="splitApprovalsMap">
  <Description>
    'replicator' attribute splitApprovalsMap ke har ek key-value pair ke liye is step ko loop chalayega.
    'lineItem' ke andar har ek loop me us individual approver ka ApprovalSet milega.
  </Description>
  <Approval action="approval" mode="parallel">
    <Arg name="approver" value="script:return itemKey;"/> <!-- 'itemKey' me approver ka naam aayega (e.g., adManager) -->
    <Arg name="approvalSet" value="ref:ownerSubSet"/>    <!-- Is approver ko sirf iska restricted data dikhega -->
    <Arg name="workItemDescription" value="string:Please approve split items assigned to you."/>
  </Approval>
  <Transition to="End"/>
</Step>

()ownerSubSet kya hai?
Original ApprovalSet ka chhota tukda (subset) jo ek specific owner ko diya gaya hai.
Example:-
Original ApprovalSet:-
ApprovalSet
 ├── RoleA
 ├── RoleB
 ├── ADGroup
 └── SAPRole

Split hua:-
Manager
   → RoleA, RoleB

AppOwner
   → ADGroup

Security
   → SAPRole

Ab:-
Manager ke liye
ownerSubSet
   ├── RoleA
   └── RoleB
AppOwner ke liye
ownerSubSet
   └── ADGroup
Security ke liye
ownerSubSet
   └── SAPRole

So:-
ownerSubSet = ek approver ka approval subset.

()lineItem practically kya karta hai?

Short answer:-
ownerSubSet = specific owner/approver ka ApprovalSet subset.
lineItem="ownerSubSet" =
Approval step me line-item approvals ke liye ownerSubSet variable use karo.

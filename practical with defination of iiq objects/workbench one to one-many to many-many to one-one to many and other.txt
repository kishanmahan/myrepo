1. One To One:-
Matlab: Ek table ka 1 record dusre table ke sirf 1 record se connected hoga.
Example:-
()users table
id    name
1----Rahul
2----Amit
()passport table
id----passport_no----user_id
1----IN123----1
2----IN456----2
Yaha:
Rahul ka ek passport
Amit ka ek passport
➡ Ek user = ek passport. 
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);

CREATE TABLE passport (
    id INT PRIMARY KEY,
    passport_no VARCHAR(50),
    user_id INT UNIQUE,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
----------------
() UNIQUE:-
lagane se ek user multiple passport nahi rakh sakta.

2. One To Many:-
Matlab: Ek table ka 1 record dusre table ke many records se connected ho sakta hai.
Example:-
()department table me:--
id----dept_name
1----IT
()employee table se:--
id----name----dept_id
1----Rahul----1
2----Amit----1
3----Neha----1
➡ Ek department me bahut employees ho sakte hain.
Ex:-
CREATE TABLE department (
    id INT PRIMARY KEY,
    dept_name VARCHAR(50)
);

CREATE TABLE employee (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES department(id)
);
------------------
3. Many To One:-
Ye basically One-To-Many ka ulta view hota hai.
Example:--
Bahut employees ek department me kaam karte hain.
()employee table:-
id   name  dept_id
1----Rahul----1
2----Amit----1
Many employees → One department. 
ex:-
CREATE TABLE employee (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES department(id)
);
-----------------------
Difference:-
Many To Many me dono side multiple hote hain, aur One To Many me sirf ek side multiple hota hai.
Example:-
Department → Employees
 1Department=many-Employes

()Lekin:-
1 Employee = sirf 1Department
IT department me many employees hain
BUT Rahul ek hi department me hai
➡ Employee multiple department me nahi ho sakta.

///////////*
()ON DELETE CASCADE ka matlab hota hai:-
Agar parent table ka data delete hoga, toh child table ka related data bhi automatically delete ho jayega.

Tumhare query me:-
FOREIGN KEY(employeeId) REFERENCES employees(employeeId) ON DELETE CASCADE
iska matlab:-
employees = parent table
employee_roles = child table
Agar employees table se koi employee delete hua, toh employee_roles me us employee ki sari matching rows automatically delete ho jayengi.

Example:-
()employees table:-
employeeId        name
E101--------------Rahul

()roles table:-
roleId	   roleName
R1----------Admin

()employee_roles table:-
employeeId	  roleId
E101------------R1

()Agar tum ye chalao:-
DELETE FROM employees
WHERE employeeId='E101';

->toh automatically:

employee_roles table me jo:-
E101 | R1 
wali row hai woh bhi delete ho jayegi.

()Same cheez:-
FOREIGN KEY(roleId) REFERENCES roles(roleId)
ON DELETE CASCADE

->Agar roles table se koi role delete hua:-
DELETE FROM roles
WHERE roleId='R1';

toh employee_roles me us role se related sari rows automatic delete ho jayengi.

()Ye mainly use hota hai:-
orphan records bachane ke liye
automatic cleanup ke liye
relationship maintain karne ke liye

Visual flow:-
employees ----<
                employee_roles
roles --------<
Parent delete → child automatic delete.

вЂ‹
1. Custom Rules (The Logic)
вЂ‹IIQ mein sabse zyada custom code Rules mein likha jata hai. Kuch main rules ye hain:
вЂ‹Correlation Rule: Account ko sahi Identity se jodne ke liye.
вЂ‹Creation Rule: Naya account bante waqt attribute set karne ke liye.
вЂ‹BuildMap Rule: Delimited files (CSV) ka data parse karne ke liye.
вЂ‹MapToResourceObject Rule: Connector se aane wale data ko modify karne ke liye.
вЂ‹Workflow Rule: Kisi specific step par custom logic chalane ke liye.
вЂ‹Policy Violation Rule: Advance SOD (Segregation of Duties) check karne ke liye.
вЂ‹2. Custom Workflows
вЂ‹Default workflows (jaise Lifecycle Manager) ke alawa aap apna khud ka Workflow Object bana sakte hain.
вЂ‹Isme aap custom Steps, Transitions, aur Approval Schemes define kar sakte hain.
вЂ‹Example: Ek aisa workflow jo provisioning ke baad kisi third-party API ko call kare.
вЂ‹3. Custom Objects (General Purpose)
вЂ‹IIQ mein ek Custom class ka object hota hai. Iska istemal aap configuration data store karne ke liye karte hain:
вЂ‹Key-Value Pairs: SQL queries, constant variables, ya mapping tables store karne ke liye.
вЂ‹Isse hardcoding se bacha ja sakta hai.
вЂ‹4. Custom Tasks (TaskDefinition)
вЂ‹Agar aapko koi background process chalani hai (jaise cleanup ya report generation), toh aap Custom Task likh sakte hain:
вЂ‹Aapko ek Java class ya Rule likhna hota hai jo TaskExecutor interface ko implement kare.
вЂ‹5. Custom Attributes (Extended Attributes)
вЂ‹Aap standard Identity ya Account schema ko extend kar sakte hain:
вЂ‹Identity Extended Attributes: Jaise employeeType, costCenter, etc.
вЂ‹Application/Link Extended Attributes: Specific application accounts ke liye extra fields.
вЂ‹6. Custom Reports
вЂ‹Agar out-of-the-box reports kaafi nahi hain, toh aap:
вЂ‹LiveReports: XML define karke custom grid aur filters bana sakte hain.
вЂ‹Plugin Reports: Java backend ka use karke advanced reporting de sakte hain.
вЂ‹7. Custom QuickLinks & UI
вЂ‹User interface ko personalize karne ke liye:
вЂ‹QuickLinks: Dashboard par naye buttons ya options add karna.
вЂ‹Dynamic Scopes: Rules ke basis par user ko dikhne wala content control karna.
вЂ‹8. Custom Connectors (Integration Config)
вЂ‹Agar koi application default connector se support nahi ho rahi:
вЂ‹Rule-Based Connectors: Poora connection logic script se likhna.
вЂ‹OpenConnector Framework (OCF): Java code ke zariye naya connector build karna.
вЂ‹9. Custom Forms
вЂ‹Provisioning ya Access Request ke waqt user se input lene ke liye aap logic-based forms design kar sakte hain jisme field validation aur visibility rules custom hote hain.

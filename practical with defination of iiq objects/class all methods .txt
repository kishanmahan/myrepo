Class ko search karta hai:-
Class ko load karta hai:-

5. Constructor.newInstance()
Kya karta hai?

Normal:

Reflection:

Example 1 — Public Constructor
import java.lang.reflect.Constructor;

public class Demo {

    public static void main(String[] args)
            throws Exception {

        Class<?> c = Student.class;

        Constructor<?> con =
                c.getConstructor(String.class);

        Object obj =
                con.newInstance("Kishan");

    }
}
Output
Public Constructor : Kishan
Explanation

Reflection ne constructor call kiya.

Example 2 — Private Constructor
import java.lang.reflect.Constructor;

public class Demo {

    public static void main(String[] args)
            throws Exception {

        Class<?> c = Student.class;

        Constructor<?> con =
                c.getDeclaredConstructor(int.class);

        con.setAccessible(true);

        Object obj =
                con.newInstance(25);

    }
}
Output:-
Private Constructor : 25
Explanation:-
Private constructor direct access nahi hota.
Isliye:-setAccessible(true) use kiya.

Fir:-
newInstance(25)
object bana diya.
newInstance(25) object return karta hai, but return type usually Object hota hai.
Example:-
Object obj = con.newInstance(25);
Ab obj ke paas actual me Student ka object hai.
Lekin variable type Object hai.

2. .class Syntax/Literal (In Java Code):-
Jab aap Java code likhte waqt kisi class ke naam ke aage .class lagate hain (jaise String.class, UserCustomizer.class), 
toh ise Class Literal kaha jata hai.
Yeh kya karta hai? Yeh us class ka ek Class object return karta hai jo memory mein load ho chuki hai.
Java mein har class jab load hoti hai, toh JVM memory mein uska ek representation metadata bana kar rakhta hai. 
.class likhne se aapko us metadata ka access mil jata hai.

Code Example:
Java
package com.project.identity;

public class Test {
    public static void main(String[] args) {
        
        // String class ka 'Class' object nikalna
        Class<?> stringClassObject = String.class;
        
        // Ab aap is object se String class ki details nikal sakte hain
        System.out.println("Class Name: " + stringClassObject.getName()); 
        // Output: java.lang.String
    }
}
(1)Class.forName() aur MyClass.class mein kya farq hai?
Dono hi aapko Class ka object dete hain, lekin:-
--(a)MyClass.class: Yeh Compile-time par check hota hai. Agar MyClass naam ki koi class aapke project mein nahi hai, 
toh code compile hi nahi hoga. Isme error aane ke chance nahi hote runtime par.
--(b)Class.forName("MyClass"): Yeh Runtime par class ko load karta hai string naam se. Agar woh class nahi mili, 
toh runtime par ClassNotFoundException aati hai.
ex:-
package com.project.identity;

public class Test {

        // String class ka 'Class' object nikalna

        System.out.println("Class Name: " + stringClassObject); 
        // Output:-Class Name: class java.lang.String
    }
}

Normal cast:-
Object obj = "Hello";
String s = (String) obj;

Reflection/Class API cast:-
String s = String.class.cast(obj);

Example:-
public class Demo {

    public static void main(String[] args) {

        Object obj = "Kishan";

        String s =
            String.class.cast(obj);

        System.out.println(s);

    }

4. getDeclaredClasses()
Kya karta hai?
Class ke andar jo nested / inner classes declare hui hain unko return karta hai.

-->Example:

class Student {

    class Address {

    }

    static class Info {

    }

}

Code:-
public class Demo {

    public static void main(String[] args) {

        Class<?> c =
            Student.class;

        Class<?>[] arr =
            c.getDeclaredClasses();

        for(Class<?> x : arr)
        {
            System.out.println(
                x.getName()
            );
        }

    }
}
Output:-
Student$Address
Student$Info

identityName vs Link Name: context.getObjectByName() kisi object ko uske Unique Name se dhoondta hai.
Identity ka name uska username/employee ID hota hai (jaise john.doe), isliye Identity.class ke saath identityName pass karna ekdum sahi hai.
Lekin SailPoint ke database mein Link (Account) ka name identityName nahi hota! 
Link ka name aamtaur par target system ka unique identifier hota hai (jaise AD ka Distinguished Name: CN=John Doe,OU=Users,DC=domain,DC=com) ya fir ek random hex ID. 
Isliye identityName se search karne par SailPoint ko koi Link nahi milega, aur link variable hamesha null rahega.
Multiple Links ka Confusion: Ek user ke paas kai saare links ho sakte hain (AD, HR, SAP, GitHub, etc.). 
Agar aap seedha Link.class dhoondoge, toh SailPoint ko kaise pata chalega ki aapko sirf AD ka hi link chahiye?

1. Loop Type Conflict (for(Link link:links))
Aapne loop likha hai: for(Link link:links) aur uske andar kiya hai linkk=(Link)link;.
Galti: Java/Beanshell mein idn.getLinks() hamesha ek raw List return karta hai jisme elements generic Object type ke hote hain. 
Agar aap direct Link link likhoge toh casting conflict ki wajah se error aa sakta hai.
Fix: Loop ko for(Object link : links) likhna sabse safe aur best practice hai, 
kyunki aap loop ke andar use waise bhi (Link) mein cast kar hi rahe ho.

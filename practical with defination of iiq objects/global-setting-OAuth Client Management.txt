--Access Token Expiration In Seconds:-
Jab koi client login karta hai, toh use ek "Token" milta hai jo ek chabi (key) ki tarah kaam karta hai.
Yeh token kitni der tak valid rahega? Default 1200 seconds (20 minutes) hota hai. Iske baad client ko fir se authenticate karna padega.

()SailPoint IdentityIQ mein Token Settings ka tab mukhya roop se tab use hota hai
jab aap Identity Authentication ke liye JSON Web Tokens (JWT) ka istemal karte hain. 
Yeh settings tay karti hain ki bahar se aane wala token kitna bharosemand hai aur use system mein kis identity se link karna hai.

1. Access Token Authentication Scope:-
Kya hai?: Yeh batata hai ki token ke paas system ke andar kya-kya karne ka "adhikaar" (permission) hai.
Kaise kaam karta hai?: Ismein aap values jaise read, write, ya admin set karte hain. 
Agar aane wale token ka scope isse match nahi karta, toh SailPoint request ko block kar dega.

2. Access Token Authentication Audience:-
Kya hai?: Yeh woh system ya service hai jiske liye token issue kiya gaya hai.
Kyun zaroori hai?: Yeh ek security layer hai jo "Token Misuse" ko rokti hai. 
Agar token kisi dusre app ke liye bana hai aur koi use IdentityIQ par chalane ki koshish karega, 
toh audience mismatch ki wajah se authentication fail ho jayega.

3. Access Token Authentication Issuers:-
Yeh setting suraksha (security) ka ek bahut mahatvapurn hissa hai jo token ke "Source" yaani ushe banane wale system ko verify karti hai.
Maqsad (Purpose): Jab aapka SailPoint system kisi external system (jaise Azure AD, Okta, ya PingFederate) se token receive karta hai, 
toh woh sabse pehle yeh dekhta hai ki kya yeh token kisi "Trusted" (bharosemand) jagah se aaya hai.
Kaise Kaam Karta Hai: Har Identity Provider (IdP) jab token banata hai, toh usmein apna ek identifier dalta hai (jise iss claim kehte hain). 
Agar aapne yahan woh issuer URL ya naam likha hai, tabhi SailPoint us token ko valid manega.
Example: Agar aap Azure AD use kar rahe hain, toh issuer URL kuch aisa hoga: [https://sts.windows.net/tenant-id/](https://sts.windows.net/tenant-id/). 
Agar token mein issuer match nahi hota, toh authentication fail ho jayega.

4. Correlation Variable:-
Kya hai?: Yeh token ke andar ki information ko SailPoint ki Identity se "match" (link) karne ka tarika hai.
Kaise kaam karta hai?:
Maan lijiye token ke andar ek field hai sub: "jatin_sharma".
Agar aapne Correlation Variable mein name set kiya hai, toh SailPoint Identity Warehouse mein jaakar check karega ki kis user ka name (User Name) "jatin_sharma" hai.
Match hone par, system samajh jayega ki yeh token Jatin ka hai aur use uski permissions ke sath login karne dega.

()SCIM kya hai?
--SCIM ka main kaam Provisioning aur Deprovisioning through automation hai.

1. SCIM Kyun Zaruri Hai? 
Pehle ke time mein, agar kisi company ko SailPoint se 50 alag-alag cloud apps (jaise Slack, Zoom, AWS, GitHub) 
ko connect karna hota tha, toh har app ka apna alag API hota tha. Developer ko har app ke liye alag code likhna padta tha.

2. SCIM ka fayda: Ab agar saari apps SCIM standard follow karti hain, toh SailPoint ko sirf ek "SCIM Connector" ki zarurat hai, 
aur woh sabse baat kar payega.

3. SCIM Kaise Kaam Karta Hai?
SCIM do main cheezon par base hota hai:-

1.JSON (Data Format): Data exchange karne ke liye JSON format ka use hota hai jo light aur fast hota hai.

2.REST (Protocol): Yeh standard HTTP methods use karta hai:-
--POST: Naya user create karne ke liye.
--GET: User details fetch/read karne ke liye.
--PUT/PATCH: User details update karne ke liye.
--DELETE: User ko remove ya deactivate karne ke liye.

3. SCIM ke Main Components:-
(a)Resources: SCIM do main resources ko samajhta hai: Users (employees/identities) aur Groups (roles/entitlements).

(b)Schema: Yeh define karta hai ki ek user ke paas kya-kya attributes honge (jaise userName, emails, displayName).

4. SailPoint IdentityIQ mein SCIM ka Use:-
--SailPoint mein jab aap SCIM connector use karte hain, toh aap niche di gayi cheezein achieve karte hain:

--Automation: Jaise hi HR system mein naya employee join hota hai, SailPoint SCIM ke zariye automatic uska account Salesforce ya Zoom mein bana deta hai.

--Security: Jab koi employee resign karta hai, toh SCIM ke through turant saare cloud accounts ek saath delete ya disable ho jaate hain.

--Compliance: Sabhi applications ek hi standard follow karti hain, isliye auditing aasaan ho jati hai.

🟢 CHAPTER 5: Correlation Rule

Purpose:-
Account ko correct identity se match karna.

Use case:-
If username format different hai.

Example logic:-
Email match
Employee ID match
Custom attribute match

🟢 CHAPTER 6: Before Provisioning Rule
Execute hota hai:-
Provisioning plan execute hone se pehle.
Use:-
Modify plan
Add attribute
Remove attribute
Conditional logic

🟢 CHAPTER 7: After Provisioning Rule
Execute hota hai:-
Provisioning complete hone ke baad.
Use:-
Logging
Notification trigger
Custom audit action

🟢 CHAPTER 8: Identity Trigger Rule
Run hota hai:-
Identity attribute change pe.
Example:-
Department change → auto role assign

🟢 CHAPTER 9: Field Value Rule
Dynamic value generate karta hai.
Example:-

firstName + lastName

🟢 CHAPTER 10: Validation Rule


Jab koi ApprovalRequest create hota hai, usme ek ya multiple ApprovalItem ho sakte hain
Matlab: ApprovalRequest = container, ApprovalItem = individual items.

()RequestDefinition define karta hai ApprovalItems template
Jab Request generate hota hai → ApprovalItems object automatically create hote hai
Each ApprovalItem track karta hai approver aur state (Pending / Approved / Rejected)
Request state update hota hai jab saare required ApprovalItems approve ho jate hai
✅ Matlab: RequestDefinition me ApprovalItem rules define hote hai, Request me ApprovalItems actual runtime objects hote hai.
------------------------------------------------------------------------------------------------------------------------------------------


🔹 ApprovalItem kya hai?
ApprovalItem = kisi specific object/action ke liye approval ka ek unit

👉 Matlab:-
Ye individual item hota hai jisko approve/reject karna hota hai
Ye kisi specific cheez (object/task) se related hota hai

🔥 Easy samajh:-
Socho tumhare paas ek request aayi:-
👉 User ko 3 cheez ka access chahiye:-
Application A
Application B
Application C

Toh system kya karega?

Uske andar 3 ApprovalItem honge:-
ApprovalItem	Kya represent karta hai
Item 1	         Application A access
Item 2	         Application B access
Item 3	         Application C access

🔹 ApprovalItem kya karta hai?
👉 ApprovalItem ek individual approval unit hai.
👉 Ye decide karta hai:-
“Kis object pe kya action hona hai aur uska approval kya status hai”

🔥 ApprovalItem ke main kaam (Functions):-
1️⃣ Target object ko represent karta hai:-
👉 Kaun sa object approve hona hai
Example:-
Identity, Role, Account, Entitlement

2️⃣ Action define karta hai:-
👉 Object ke saath kya karna hai
Example:-
ADD, REMOVE, UPDATE, DELETE, ENABLE, DISABLE
👉 Ye actual change nahi karta, sirf instruction deta hai

3️⃣ Approval status track karta hai:-
👉 Current state kya hai
PENDING → APPROVED → REJECTED

4️⃣ Approvers manage karta hai:-
👉 Kaun log approve karenge
Manager, Admin, Security Team

5️⃣ Comments store karta hai:-
👉 Approver ne kya reason diya
"Looks good", "Not allowed"

6️⃣ Approval decision handle karta hai:-
👉 Approver action leta hai
item.approve();
item.reject();

7️⃣ Audit aur tracking karta hai:-
👉 Ye sab track karta hai:-
Kab create hua
Kisne create kiya
Kisne modify kiya
Kab approve hua

8️⃣ ApprovalRequest ka part hota hai:-
👉 Ye individual item hai jo ek request ke andar hota hai

   ├── ApprovalItem 1
   ├── ApprovalItem 2
   
9️⃣ Workflow trigger me role play karta hai:-
👉 ApprovalItem ke basis pe:-
Workflow aage badhta hai
ProvisioningPlan generate hota hai
Actual system update hota hai

1️⃣ getTargetObject() & setTargetObject(SailPointObject obj)
Kaam:-
ApprovalItem me kaunsa SailPoint object approve/reject karna hai, usko set aur get karta hai.

Example:-
import sailpoint.api.SailPointContext;
import sailpoint.api.SailPointFactory;
import sailpoint.object.Identity;
import sailpoint.object.ApprovalItem;

public class ApprovalItemExample {
    public static void main(String[] args) throws Exception {
        SailPointContext context = SailPointFactory.getCurrentContext();

        // Identity fetch
        Identity user = context.getObjectByName(Identity.class, "kishan");

        // ApprovalItem create
        ApprovalItem item = new ApprovalItem();
        item.setTargetObject(user);  // target object set

        // Get target object
        Identity target = (Identity) item.getTargetObject();
        System.out.println("ApprovalItem target object: " + target.getName());
    }
}
Output:-
ApprovalItem target object: kishan

🔹 1️⃣ approve():-
✔️ Kaam:-
👉 ApprovalItem ko APPROVED state me daal deta hai

Status change hota hai → APPROVED
Workflow aage badhta hai
Final provisioning (actual change) trigger ho sakta hai
Example:
ApprovalItem item = new ApprovalItem();

item.setStatus("PENDING");

// approver approve karta hai
item.approve();

System.out.println(item.getStatus());
Output:-
APPROVED
----------------------------------------------------------------------
🔹 2️⃣ reject():-
✔️ Kaam:-
👉 ApprovalItem ko REJECTED state me daal deta hai

Status change hota hai → REJECTED
Workflow yahi ruk jata hai
Koi change system me apply nahi hota
Example:
ApprovalItem item = new ApprovalItem();

item.setStatus("PENDING");

// approver reject karta hai
item.reject();

System.out.println(item.getStatus());
Output:-
REJECTED
🔥 Real Flow:-
PENDING → REJECTED → No change applied

🔹 ❓ ApprovalItem ko kaun approve karta hai?
👉 Approver(s) approve karte hain,
Ye ho sakte hain:-
👨Manager (identity ka manager)
🛡️Application Owner
👨Admin / Security Team
👥Custom approver (workflow me define hota hai)
👉 Ye sab tu setApprovers() me define karta hai ya workflow decide karta hai.
-------------------------------------------------------------------------------
🔹 ❓ Ek baar me approve hota hai ya ek-ek karke?

👉 ⚠️ Answer: Dono possible hai (workflow pe depend karta hai)

🔸 Case 1: Single Approver
PENDING → Manager approve → DONE

👉 Sirf ek banda approve karega

🔸 Case 2: Multiple Approvers (ANY / OR logic)
Manager OR Admin me se koi ek approve kare → DONE

👉 Ek ne approve kiya → kaam ho gaya

🔸 Case 3: Multi-Level Approval (Sequential 🔥)
Step 1: Manager approve
Step 2: Admin approve
Step 3: Security approve

👉 Ek-ek karke approve hota hai

🔸 Case 4: Parallel Approval (ALL required)
Manager ✔️
Admin ✔️
Security ✔️

👉 Sabko approve karna padega.

🔥 Short Answer (yaad rakhna):-
👉 Approve actually ApprovalItem hota hai
👉 ApprovalRequest bas container / wrapper hota hai

🔹 Detail me samajh:-
🧱 Structure kya hai?
ApprovalRequest
   ├── ApprovalItem 1
   ├── ApprovalItem 2
   ├── ApprovalItem 3

👉 Matlab:-
ApprovalRequest = poora request
ApprovalItem = individual kaam (real approval yahi hota hai)

🔹 Approval kaun hota hai?
👉 ✅ Approver ApprovalItem ko approve karta hai,
item.approve();
👉 Ye actual approval hai

🔹 Fir ApprovalRequest ka kya role hai?
👉 Ye bas:-
multiple items ko hold karta hai
overall status track karta hai

🔥 Example Flow:-
Step 1: Request create
ApprovalRequest (PENDING)
   ├── Item 1 (PENDING)
   ├── Item 2 (PENDING)
Step 2: Approver action
item1.approve();
item2.approve();
Step 3: Final Result
ApprovalRequest → APPROVED (jab sab items approve ho jaye)

🔥 Important Logic:-
Situation	            Result
Sab items APPROVED----Request APPROVED
Koi ek REJECTED-------Request REJECTED
Kuch pending----------Request PENDING

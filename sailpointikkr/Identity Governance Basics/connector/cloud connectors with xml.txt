()Cloud connectors kya hai?
--Cloud Connector = Integration component jo IAM system ko Cloud applications se connect karta hai.
Matlab:-
IAM tool (jaise SailPoint IdentityNow, Okta, Microsoft Entra ID)
->Connector->
Cloud Application (jaise Salesforce, Slack, Workday)
Connector ka kaam hota hai user account aur access ko manage karna.

()Cloud Connector kya kya karta hai?
1️⃣ Account Provisioning:-
User create karta hai cloud application me.
Example:-
IAM → Salesforce user create
User: Rahul
App: Salesforce
Action: Create Account

2️⃣ Account Deprovisioning:-
User ka access remove karta hai.
Example:-
Rahul leaves company
IAM → Salesforce account disabled

3️⃣ Aggregation:-
Cloud application se user data aur entitlements IAM me import karta hai.
Example:-
Salesforce Users
Slack Users
GitHub Users
IAM me sync ho jate hain.

4️⃣ Password Management:-
Connector password change bhi kar sakta hai.
Example:-
IAM password reset
↓
Salesforce password reset

5️⃣ Entitlement Management:-
Application roles / permissions manage karta hai.
Example:-
Slack Role → Admin
Salesforce Role → Sales User

()Types of Connectors:-
1️⃣ Direct Connector:-
IAM directly connect karta hai application se.
Example:-
IAM → Salesforce API

2️⃣ Gateway Connector:-
Firewall ke andar applications ke liye gateway use hota hai.
Example:-
IAM Cloud
     ↓
Virtual Appliance
     ↓
On-Prem Application

3️⃣ API Based Connector:-
REST API use karta hai.
Example:-
POST /users
GET /accounts
DELETE /users

1️⃣1️⃣ Add Entitlement
Kya karta hai

User ko permission / role assign karta hai.

Code
import java.util.HashMap;
import java.util.ArrayList;
import java.util.Map;
import java.util.List;

class AddEntitlement {

    static Map<String,List<String>> access = new HashMap<String,List<String>>();

    public static void add(String user,String role)
    {

        if(!access.containsKey(user))
        {
            access.put(user,new ArrayList<String>());
        }

        access.get(user).add(role);

        System.out.println(role + " added to " + user);

    }

    public static void main(String[] args) {

        add("john","Admin");

    }
}
Output
Admin added to john

1️⃣2️⃣ Remove Entitlement
Kya karta hai?
User se permission remove karta hai.

import java.util.HashMap;
import java.util.ArrayList;
import java.util.Map;
import java.util.List;

class RemoveEntitlement {

    static Map<String,List<String>> access = new HashMap<String,List<String>>();

    public static void remove(String user,String role)
    {

        if(access.containsKey(user))
        {
            access.get(user).remove(role);
        }

        System.out.println(role + " removed from " + user);

    }

    public static void main(String[] args) {

        remove("john","Admin");

    }
}
Output
Admin removed from john

()Connector XML configuration:-
--1️⃣ Basic Connector XML Configuration
<Application name="Salesforce" type="WebService">

    <!-- Connection Configuration -->
    <Attributes>
        <Map>
            <entry key="apiUrl" value="https://api.salesforce.com"/>
            <entry key="username" value="admin@company.com"/>
            <entry key="password" value="password123"/>
            <entry key="clientId" value="abc123xyz"/>
            <entry key="clientSecret" value="secret123"/>
        </Map>
    </Attributes>

    <!-- Schema Configuration -->
    <Schemas>
        <Schema name="account" type="account">
            <AttributeDefinition name="username" type="string"/>
            <AttributeDefinition name="email" type="string"/>
            <AttributeDefinition name="department" type="string"/>
            <AttributeDefinition name="status" type="string"/>
        </Schema>
    </Schemas>

</Application>

🔹1️⃣ IdentityService kya hai?
IdentityService = SailPoint SDK / API ka service class
Purpose: Identity objects ke upar CRUD operations aur business logic handle karna
Basically, Identity ke liye central service layer
Ye directly Identity objects ko create, update, delete, search, ya query karne ke liye use hota hai.

🔹2️⃣ IdentityService ka kaam:-
Kaam	                                             Explanation
Create/Update/Delete Identity:----------------Nayi identity create karna, existing update karna, ya delete karna
Search/Lookup Identity:-----------------------Identity naam / email / id se find karna
Manager assignment:---------------------------Identity ke manager set ya update karna
Role/Entitlement assignment:------------------Identity ko roles aur entitlements assign karna
Certification/Workflow support:---------------Identity ke upar certifications ya triggers run karna
Validation:-----------------------------------Identity ke data ko validate karna before save
Batch operations:-----------------------------Multiple identities ko ek sath handle karna

🔹 4️⃣ DB me relation:-
IdentityService Identity object ko DB me persist karne ka layer hai
Example:-
Identity → main table (id, name, email, status, department, manager_id)
Identity_Accounts → linked accounts
Identity_Roles → roles
Identity_Entitlements → permissions
Service methods internally SailPointContext.saveObject(identity) aur commitTransaction() ka use karte hain.

()createIdentity(Identity identity):-
Kya karta hai: Nayi identity create karta hai aur DB/context me save karta hai
Example / Output:-

newUser.setName("Ravi Kumar");
newUser.setEmail("ravi.kumar@company.com");
newUser.setStatus("Active");

()1️⃣ triggerCertifications(Identity identity):-
Kya karta hai:-
Identity ke liye Certification campaign / review process start karta hai
Example: Manager ko notification send hoti hai, reviewer ko approval task create hota hai
Ye directly Identity ke assigned certifications ko initiate karta hai
Example / Output:
Identity user = identityService.getIdentityByName("Ravi Kumar");
identityService.triggerCertifications(user);

System.out.println(identityService.getCertificationStatus(user)); 
// Output: PENDING_REVIEW
Flow:-
Certification trigger hota hai
Reviewers ko tasks assign hote hain
Status change hota hai: PENDING_REVIEW → APPROVED / REJECTED


🔹 1️⃣ triggerCertifications(Identity identity) – EXACT kya hota hai
❓ Tera sawal:-
Kya isse sare users review me jate hain?
Naya campaign banta hai ya existing chalta hai?

✅ Correct samajh (IMPORTANT):-
👉 Ye method sirf ek specific Identity (user) ke liye certification trigger karta hai
👉 Saare users nahi jate review me ❌

🔥 Actual Flow:-
triggerCertifications(user)
        ↓
System check karta hai CertificationDefinition
        ↓
Us user ke liye CertificationEntity create hota hai
        ↓
Uske access ke liye CertificationItems bante hain
        ↓
Reviewer (manager) ko task milta hai
🧠 Deep Samajh:-
🔸 1. Kya naya CertificationCampaign banta hai?
❌ Direct nahi
✅ Existing CertificationDefinition use hota hai
👉 But:-
System ek runtime campaign instance generate kar sakta hai internally

🔸 2. Kya sare users jate hain?
❌ Nahi
✅ Sirf wo Identity (user) jisko pass kiya hai
identityService.triggerCertifications(ravi);
👉 Sirf Ravi review me jayega, pura company nahi

🔸 3. Kya banta hai internally?
Object	               Kya hota hai
CertificationEntity:---Ravi ke liye
CertificationItem:-----Ravi ke roles / entitlements
Reviewer:--------------Ravi ka manager

🎯 Example:-
identityService.triggerCertifications(user);
System.out.println(identityService.getCertificationStatus(user));
Output:-
PENDING_REVIEW
👉 Matlab:-
Certification start ho gaya
Manager ko review task mil gaya

2️⃣ runIdentityTriggers(Identity identity):-
Kya karta hai:-
Identity me defined triggers execute karta hai
Trigger = predefined action or workflow (jaise manager assignment, role provisioning, account creation)
Example: Jab identity create hoti hai, default roles ya entitlements automatically assign ho jate hain
Example / Output:
identityService.runIdentityTriggers(user);

System.out.println(user.getRoles()); 
// Output: [Developer, IT_Admin] (Assigned automatically by trigger)
Flow:-
Identity triggers load hote hain (from IdentityDefinition or system config)
Sab triggers sequentially execute hote hain
Result: Roles/Entitlements/Notifications applied automatically

3️⃣ getCertificationStatus(Identity identity):-
Kya karta hai:-
Identity ke certification process ka current status return karta hai
Status values typically: NOT_STARTED, PENDING_REVIEW, APPROVED, REJECTED
Example / Output:-
String status = identityService.getCertificationStatus(user);
System.out.println(status); 
// Output: APPROVED
Use case:-
Compliance team ya admin check kar sakta hai ki identity ke certifications complete hue ya pending hain.

1️⃣ validateIdentity(Identity identity):-
Kya karta hai:
Identity ke fields aur business rules validate karta hai
Check karta hai: required fields, email format, status, manager assignment, role conflicts etc.
Agar invalid data hai → exception throw karta hai ya validation error return karta hai
Example / Output:
try {
    identityService.validateIdentity(user);
    System.out.println("Validation Passed");
} catch (ValidationException e) {
    System.out.println("Validation Failed: " + e.getMessage());
}
// Output: Validation Passed

🔍 Ye kya kya validate karta hai:-
1️⃣ Required Fields Check:-
Name empty nahi hona chahiye
Email required
Status required
Name = null ❌

2️⃣ Email Format Check:-
ravi@company.com ✅
ravi.company.com ❌

3️⃣ Status Validation:-
Allowed:-
Active
Inactive
Suspended
Deleted ❌

4️⃣ Manager Validation:-
Manager exist hona chahiye
Self manager nahi hona chahiye
Ravi → Ravi (manager) ❌

5️⃣ Role Conflict (SoD – Segregation of Duties):-
Example:
Admin + Auditor ❌
👉 Security risk

6️⃣ Entitlement Validation:-
Duplicate nahi hona chahiye
Invalid entitlement nahi hona chahiye

7️⃣ Account Validation:-
Same system ke duplicate accounts nahi
Invalid account link nahi

8️⃣ Custom Business Rules:-
Company specific:-
IT role → IT department
Finance access → approval required
--------------------------------------------------------------------------------------------------------------------------
2️⃣ isValidEmail(Identity identity):-
Kya karta hai:
Identity ka email valid format me hai ya nahi check karta hai
Invalid email → false return karta hai
Example / Output:
user.setEmail("ravi.kumarcompany.com"); // Invalid email
System.out.println(identityService.isValidEmail(user)); 
// Output: false

user.setEmail("ravi.kumar@company.com"); // Correct email
System.out.println(identityService.isValidEmail(user)); 
// Output: true

3️⃣ isValidStatus(Identity identity);-
Kya karta hai:
Identity ka status check karta hai
Valid values: "Active", "Inactive", "Suspended"
Agar invalid status → false return karta hai
Example / Output:
user.setStatus("Active");
System.out.println(identityService.isValidStatus(user)); 
// Output: true

user.setStatus("Deleted");
System.out.println(identityService.isValidStatus(user)); 
// Output: false

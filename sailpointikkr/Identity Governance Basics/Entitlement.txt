🔥 Entitlement kya hota hai?
👉 Entitlement = actual permission / access right
Matlab: user system me kya kar sakta hai.

1️⃣ Entitlements (हक / अधिकार):-
Ye basically user ke paas jo access ya permissions hai kisi application ya system me, usko represent karta hai.
Example:-
User ko SAP system me read-only access diya gaya → ye ek entitlement hai.
User ko Finance folder me write permission → ye bhi ek entitlement hai.
SailPoint me entitlements ko Identity -> Links -> Accounts -> Entitlements ke through track kiya jata hai.


🔥 Entitlement vs Role vs AccessProfile:-
Concept	                    Meaning
Entitlement:-----------actual permission (READ, WRITE)
AccessProfile:---------entitlements ka group
Role:------------------multiple AccessProfiles ka group.

🔥 Real System Example (SailPoint):-
👉 Application: SAP
Entitlements ho sakte hain:-
SAP_USER_CREATE
SAP_USER_DELETE
SAP_REPORT_VIEW

🔥 Entitlement ke andar kya hota hai?
👉 Entitlement sirf ek string nahi hota
👉 Ye ek object hota hai jisme multiple fields/properties hoti hain

()👉 Type set karne se system ko pata chalta hai ki is entitlement ko kaise treat karna hai
Matlab:-kaise process hoga + kahan apply hoga + kya effect hoga.

🔥 Type set karne se system kya decide karta hai?
1️⃣ Provisioning kaha hoga:-
Type	        Target
Permission:----DB / API
Group:--------Active Directory
Role:----------Application
License:-------SaaS system

2️⃣ Kaunsa operation chalega:-
👉 Example:-
Group → user add/remove
Permission → grant/revoke
License → assign/unassign

3️⃣ Decide karta hai — kaunsa connector use hoga:-
👉 Example:-
AD connector
DB connector
API connector

3️⃣ Rule kaise apply hoga:-
Type	        Rule Behavior
Transaction:---business rules
Data:----------filter rules
UI:------------visibility rules
License:-------eligibility rules

4️⃣ Approval ka level:-
👉 Example:-
Privilege → High approval
Normal Permission → Low approval

5️⃣ UI behavior:-
👉 Example:-
UI type → menu show/hide
Data type → filtered data

🔥 Real Flow (System internally):-
Entitlement Type set
   ↓
System identify karta hai
   ↓
Provisioning plan banata hai
   ↓
Correct connector call hota hai
   ↓
Target system me Access apply hota hai

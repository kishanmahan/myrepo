()Access Request kya hai?
--Access Request = user kisi application ka access maangta hai.
Access Profile ek permission bundle hota hai, 
jabki Access Request wo process hai jisme user us Access Profile ka access maangta hai.
👉 Jab koi user access maangta hai (role / access profile / entitlement)
👉 Tab ek AccessRequest object create hota hai

Access Request ka use:-
1️⃣ Application access lene ke liye
User kisi bhi app ka access request karta hai.
Example;-
Salesforce
ServiceNow
Jira
SAP

2️⃣ Role access lene ke liye:-
Example:-
HR Role
Finance Role
Developer Role

3️⃣ Temporary access lene ke liye;
Example:-
Developer ko 2 din ke liye production access chahiye.

7️⃣ Access Profile:-
Kya hai?
Access Profile = Application access ka bundle / package.
Isme entitlements (permissions) group kiye jate hain taaki ek hi request me access mil jaye.
Short Answer:-
👉 Access Profile में user application के entitlements (permissions / groups / roles) के लिए request करता है।
Access Profile kyu bhejte (request) hai?
Access Profile isliye bhejte hain taaki user ko kisi application ka access mil sake.

()Access Profile Important Attributes:-
Attribute	Kya karta hai
name------------Access profile ka naam
source----------Application source
entitlements----Permissions list
requestable-----User request kar sakta hai ya nahi
description-----Access ka purpose

()Access Profile में क्या-क्या होता है:-
Access Profile के अंदर:-
Entitlements
Approval workflow
Owner
Description
Source (Application)

()Example:-
Access Profile:-
HR App Access

Inside Access Profile:-
Entitlement 1 → HR_Read
Entitlement 2 → HR_Report

अगर user HR App Access request करेगा तो दोनों entitlements मिल सकते हैं।

()IdentityNow Access Request Flow:-
User
↓
Access Request
↓
Access Profile choose
↓
Manager / Owner approval
↓
Provisioning
↓
Application में access create

()🔥 validate() kya hota hai?
👉 validate() ka kaam hota hai:-
Check karna ki AccessProfile sahi bana hai ya nahi,
Agar kuch galat hua ❌
👉 Error throw karega (Exception)
Agar sab sahi hai ✅
👉 Successfully pass ho jayega.

1️⃣ Name check:-
if(name == null || name.isEmpty())
❌ Error:-
Name is required

2️⃣ Entitlements empty hai ya nahi:-
if(entitlements == null || entitlements.isEmpty())
❌ Error:-
At least one entitlement required

3️⃣ Applications linked hai ya nahi:-
if(applications == null || applications.isEmpty())
❌ Error:-
Application must be assigned

4️⃣ Owner set hai ya nahi:-
if(owner == null)
❌ Error:-
Owner is mandatory

5️⃣ Status valid hai ya nahi:-
if(!status.equals("ACTIVE") && !status.equals("INACTIVE"))
❌ Error:-
Invalid status

6️⃣ Duplicate entitlements check:-
Set<String> set = new HashSet<>(entitlements);
if(set.size() != entitlements.size())
❌ Error:-
Duplicate entitlements not allowed.

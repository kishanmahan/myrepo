()ProvisioningResult kya hai?
--
👉 Ye ek object hai jo Provisioner / Application se return hota hai
👉 Jab bhi tum koi provisioning request bhejte ho (jaise user create ya disable), to uska final result yahi store hota hai
📌 Simple language me:-
“Provisioning ka output / response = ProvisioningResult”

🔹 Ye kya karta hai?
ProvisioningResult:-
✅ Batata hai operation success hua ya fail
❌ Agar fail hua to error details deta hai
📄 Har provisioning request ka status maintain karta hai
🔗 Track karta hai ki kaunse account pe kya action hua
📊 Logging & auditing me help karta hai.

🔹 Andar kya kya hota hai? (Important fields)
ProvisioningResult ke andar usually ye sab hota hai:-

1️⃣ Status:-
SUCCESS
FAILURE
PENDING
👉 Batata hai operation ka final result

2️⃣ Errors / Exceptions
Agar provisioning fail hua
👉 to error message / exception yahan store hota hai
Example:-
Account not found
Invalid attribute
Permission denied

3️⃣ Account Information:-
Kaunse account pe provisioning hua
Application name
Native identity / account ID

4️⃣ Operation Type:-
CREATE
UPDATE
DELETE
DISABLE

5️⃣ Messages / Logs:-
Success message
Warning
Debug logs

6️⃣ Provisioning Plan reference:-
👉 Kaunsa provisioning plan execute hua uska reference

7️⃣ Attributes / Changes:-
👉 Kya change hua:-
Email update
Role assign
Department change

8️⃣ Request ID / Tracking ID:-
👉 Har provisioning request ka unique ID
👉 Troubleshooting me help karta hai.

❌ ProvisioningResult:-
ProvisioningResult object DB me store nahi hota,
👉 Ye mainly runtime object hai,
👉 Memory me exist karta hai,
👉 Direct DB table me as-is store nahi hota.

🔹 1️⃣ SUCCESS:-
👉 Provisioning successfully complete ho gaya
👉 Sab operations (create/update/delete) successfully apply ho gaye

🔹 2️⃣ FAILURE:-
👉 Provisioning fail ho gaya
👉 Koi error aaya (application issue, validation error, permission issue, etc.)

🔹 3️⃣ PENDING:-
👉 Request abhi process nahi hua hai
👉 Queue me hai ya wait kar raha hai

🔹 4️⃣ IN_PROGRESS / PROCESSING:-
👉 Provisioning chal raha hai
👉 Abhi complete nahi hua

🔹 5️⃣ PARTIAL_SUCCESS:-
👉 Kuch operations success hue, kuch fail
👉 Mixed result
Example:-
5 attributes update karne the
3 success, 2 fail

🔹 6️⃣ RETRY / RETRYING:-
👉 Fail hone ke baad system dubara try kar raha hai
👉 Temporary issue (like network / timeout)

🔹 7️⃣ NOT_APPLICABLE:-
👉 Provisioning apply hi nahi hua
👉 Rule ya condition match nahi hua
Example:-
Agar kisi user pe provisioning rule apply nahi hota

🔹 8️⃣ SKIPPED:-
👉 Ye operation intentionally skip kar diya gaya
👉 Koi rule ya condition ki wajah se

🔹 9️⃣ CANCELLED:-
👉 Provisioning request cancel ho gaya
👉 Admin ya system ne rok diya

🔹 🔟 UNKNOWN / ERROR:-
👉 Status clear nahi hai ya unexpected error aaya
👉 Debugging me use hota hai.

import sailpoint.api.SailPointContext;
import sailpoint.api.SailPointContextFactory;

import sailpoint.object.Identity;
import sailpoint.object.Application;
import sailpoint.object.ProvisioningPlan;
import sailpoint.object.ProvisioningResult;

import sailpoint.provisioning.Provisioner;

import java.util.List;
import java.util.ArrayList;

public class FullProvisioningDemo {

    public static void main(String[] args) {

        SailPointContext context = null;

        try {
            // 🔹 1. Context login
            SailPointContextFactory factory = new SailPointContextFactory();
            context = factory.getContext("spadmin", "admin123");

            // 🔹 2. Identity fetch
            Identity identity = context.getObjectByName(Identity.class, "john.doe");

            // 🔹 3. Application fetch
            Application app = context.getObjectByName(Application.class, "Active Directory");

            // 🔹 4. Plan create
            ProvisioningPlan plan = new ProvisioningPlan();
            plan.setIdentity(identity);

            // 🔹 5. AccountRequest
            ProvisioningPlan.AccountRequest accountRequest =
                    new ProvisioningPlan.AccountRequest();

            accountRequest.setApplication(app.getName());
            accountRequest.setNativeIdentity("john.doe");
            accountRequest.setOperation(
                    ProvisioningPlan.AccountRequest.Operation.Modify
            );

            // 🔹 6. AttributeRequest
            ProvisioningPlan.AttributeRequest attr1 =
                    new ProvisioningPlan.AttributeRequest(
                            "email",
                            ProvisioningPlan.Operation.Set,
                            "john@gmail.com"
                    );

            ProvisioningPlan.AttributeRequest attr2 =
                    new ProvisioningPlan.AttributeRequest(
                            "department",
                            ProvisioningPlan.Operation.Set,
                            "IT"
                    );

            ProvisioningPlan.AttributeRequest attr3 =
                    new ProvisioningPlan.AttributeRequest(
                            "role",
                            ProvisioningPlan.Operation.Add,
                            "Developer"
                    );

            // 🔹 7. Add attributes
            accountRequest.add(attr1);
            accountRequest.add(attr2);
            accountRequest.add(attr3);

            // 🔹 8. Add to plan
            plan.add(accountRequest);

            // 🔹 9. Execute
            Provisioner provisioner = new Provisioner(context);
            ProvisioningResult result = provisioner.execute(plan);

            // 🔹 10. Manual changes list (NO Arrays.asList)
            List<String> changes = new ArrayList<String>();
            changes.add("Email updated to john@gmail.com");
            changes.add("Department changed to IT");
            changes.add("Role Developer added");

            result.setChanges(changes);

            // 🔹 11. Output
            System.out.println("===== FINAL OUTPUT =====");
            System.out.println("Status: " + result.getStatus());
            System.out.println("Success: " + result.isSuccess());
            System.out.println("Errors: " + result.getErrors());
            System.out.println("Warnings: " + result.getWarnings());
            System.out.println("Changes: " + result.getChanges());
            System.out.println("RequestId: " + result.getRequestId());
            System.out.println("Application: " + result.getApplication());
            System.out.println("Account: " + result.getAccount());

        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            if (context != null) {
                context.close();
            }
        }
    }
}

()factory.getContext("spadmin", "admin123");:-

🔹 1. Ye return kya karta hai?
👉 Ye return karta hai:-
SailPointContext object
Example:-
SailPointContext context = factory.getContext("spadmin", "admin123");
📌 Matlab:-
👉 Tumhe ek authenticated session milta hai IdentityIQ ka
👉 Jisse tum DB aur SailPoint ke objects access kar sakte ho

🔹 2. Andar kya kya hota hai (Behind the Scenes):-
Jab tum ye call karte ho, internally ye steps hote hain:-

✅ Step 1: Authentication
"spadmin" username check hota hai
"admin123" password verify hota hai
👉 Agar galat hua:-
LoginException / Authentication Failed.

✅ Step 2: Session create hota hai:-
IdentityIQ ek session banata hai,
Ye session ek logged-in user session jaisa hota hai.

✅ Step 3: Context object ban jata hai:-
👉 Ye context ke andar hota hai:-
DB connection (Hibernate session)
Logged-in user (spadmin)
Permissions / roles
SailPoint APIs ka access

🔹 3. Context se kya kya kar sakte ho?
👉 Ye sab powerful kaam kar sakte ho:-
()🔸Identity fetch:-
Identity id = context.getObjectByName(Identity.class, "john.doe");
()🔸Application fetch:-
Application app = context.getObjectByName(Application.class, "HR System");
()🔸Provisioning:-
Provisioner p = new Provisioner(context);
p.execute(plan);
()🔸Query run:-
context.search(Identity.class, queryOptions);

spt_link table

()CertificationItem kya hai?
--🔹 CertificationItem kya hai?
CertificationItem SailPoint IdentityIQ me ek object hota hai jo certification(access review) ke andar ek single item/record represent karta hai.
👉 Matlab:-
Ek user ke paas 10 access hai
Toh certification me 10 CertificationItem banenge
Har item ek access ko represent karega (role / entitlement / account)

🔹 CertificationItem kya karta hai?
👉 Ye decide karta hai:-
Access approve hoga ya revoke
Reviewer ne kya decision liya
Kis user ka kaunsa access review ho raha hai
👉 Simple:-
"Ek access ka review record = CertificationItem"
----------------------------------------------------------------
🔹 CertificationItem ke andar kya hota hai (Fields):-
Main important fields 👇:-
🔸 1. Basic Info
id → unique ID (DB me primary key)
type → item ka type (ROLE / ENTITLEMENT / ACCOUNT)
name → access ka naam
description → details

🔸 2. Identity Info
identityId → user ka ID
identityName → user ka naam
👉 Kaunse user ka access review ho raha hai

🔸 3. Application Info:-
application → kis system se access hai (SAP, AD, etc.)
accountId → user ka account id

🔸 4. Access Info:-
value → actual entitlement value
displayValue → UI me dikhne wala naam

🔸 5. Decision / Action Fields:-
decision → reviewer ka decision
APPROVE
REVOKE
DELEGATE
recommendation → system kya suggest kar raha hai
comments → reviewer ka comment

🔸 6. Dates:-
created → kab bana
modified → kab update hua

🔸 7. Reviewer Info:-
reviewer → kisne review kiya
owner → access ka owner

🔸 8. Flags:-
exception → rule break hua kya
remediated → revoke ho gaya kya.
--------------------------------------------------------------------------------------------
🔹 CertificationItem ke Types:-
👉 type field me aate hain:-

------------------------------------------------
🔹 CertificationItem ke States (bahut important 🔥):-
👉 Ye batata hai item ka current status kya hai.
Main states:-
Open:-
Abhi review pending hai

Approved:-
Access allow ho gaya

Revoked:-
Access hata diya gaya

Mitigated:-
Risk accept kar liya (remove nahi kiya)

Delegated:-
Kisi aur ko bhej diya review ke liye

Expired:-
Time over ho gaya

Completed:-
Final ho gaya (approve/revoke)

()🔹 1️⃣ recommendation kya hai?
👉 Meaning:-
Recommendation = System ka suggestion
👉 System (rules / policy / analytics) ye batata hai:
“Is access ko rakhna chahiye ya hata dena chahiye”

🔸 Example:-
item.setRecommendation("REVOKE");

👉 Matlab:-
System bol raha hai → ❌ hata do access

🔸 Possible values:-
APPROVE → rakh lo access
REVOKE → hata do
NONE → koi suggestion nahi

🔸 Real Use:-
👉 Jab reviewer dekhta hai UI me:-
User: Ravi
Access: SAP Admin
Recommendation: REVOKE ⚠️

👉 Reviewer sochta hai:-
System bol raha hai risky hai
Main approve karu ya revoke?

🔹 2️⃣ owner kya hai?
👉 Meaning:-
Owner = Access / Entitlement ka malik
👉 Wo banda jo decide kare:-
“Ye access kis-kis ko milna chahiye”

🔸 Example:-
item.setOwner("Manager1");

👉 Matlab:-
Manager1 is access ka owner hai.

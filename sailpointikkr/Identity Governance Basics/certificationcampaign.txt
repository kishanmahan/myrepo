()CertificationCampaign kya hai?
👉CertificationCampaign = ek poora Access Review Campaign,
Simple:-
Ye ek top-level object hai jo define karta hai:-
kaun review karega
kya review hoga (role/Entitlements/permissions)
kab tak hoga
CertificationCampaign = poora system jo access review ko manage karta hai (start → review → complete → audit)

🧠 Real Example:-
Company ne bola:-
👉 "Saare employees ke access ka review karo 30 din me"
→ Ye poora campaign = CertificationCampaign

🔥 Ye kya karta hai?
👉 Main kaam:-
✅ 1. Access Review start karta hai
✅ 2. Users ko assign karta hai reviewers ko
✅ 3. Deadlines manage karta hai
✅ 4. Progress track karta hai
✅ 5. Final result store karta hai.

🔥 Andar kya hota hai?
CertificationCampaign
 ├── Id
 ├── Name
 ├── Type
 ├── Status
 ├── Start Date
 ├── End Date
 ├── List<Certification>   ← users ka review 🔥

🔹 Certification kya hota hai?
👉
Certification = ek reviewer ke liye assigned review
Certification
 ├── Reviewer (Manager)
 ├── List<AccessReviewItem>

🔥 DB me store hota hai kya?
👉 ✅ YES — 100% DB me save hota hai.

🗄️ Database Structure:-
CertificationCampaign  (table)
   ↓
Certification         (table)
   ↓
AccessReviewItem      (table)
   ↓
AccessReviewDecision  (table)

💡 Relation:-
1 Campaign → Many Certifications  
1 Certification → Many Review Items  
1 Item → Many Decisions  
------------------------------------------------------------------------------------------------------------
🔥 CertificationCampaign ke Types:-
1️⃣ Manager Certification:-
👉 Manager apne team ka review karta hai

2️⃣ Application Owner Certification:-
👉 App owner access review karta hai

3️⃣ Role Certification:-
👉 Roles ka review hota hai

4️⃣ Entitlement Certification:-
👉 Permissions ka review

5️⃣ Identity Certification:-
👉 Full user access review

6️⃣ Targeted Certification:-
👉 Specific users / access ka review
-------------------------------------------------------------------------------------------------------
🔥 Status (VERY IMPORTANT 🔥):-
🟡 1. Staged:-
👉 Campaign create hua hai, start nahi hua

🔵 2. Active:-
👉 Review chal raha hai

🟠 3. Completed:-
👉 Sab reviewers ne kaam khatam kar diya

🔴 4. Closed:-
👉 Campaign officially close ho gaya

⚫ 5. Cancelled:-
👉 Beech me band kar diya

🟣 6. Expired:-
👉 Deadline cross ho gayi

import sailpoint.api.SailPointContext;
import sailpoint.api.SailPointFactory;
import sailpoint.object.CertificationCampaign;
import sailpoint.object.CertificationItem;
import sailpoint.object.Identity;

import java.util.ArrayList;
import java.util.Date;
import java.util.List;

public class AccessReviewFullDemo {

    public static void main(String[] args) throws Exception {

        // 🔹 1. Get SailPoint context
        SailPointContext context = SailPointFactory.getCurrentContext();

        // 🔹 2. Create new Access Review / CertificationCampaign
        CertificationCampaign campaign = new CertificationCampaign();
        campaign.setName("Quarterly Finance Access Review");
        campaign.setDescription("Finance roles access review");
        campaign.setStatus("OPEN");
        campaign.setCreated(new Date());
        campaign.setModified(new Date());

        // 🔹 3. Set Owner
        Identity owner = context.getObjectByName(Identity.class, "IAM_Admin");
        campaign.setOwner(owner);

        // 🔹 4. Set Violation Owner
        Identity violationOwner = context.getObjectByName(Identity.class, "manager1");
        campaign.setViolationOwner(violationOwner);

        // 🔹 5. Set Reviewer
        Identity reviewer = context.getObjectByName(Identity.class, "manager2");
        campaign.setReviewer(reviewer);

        // 🔹 6. Set Items
        List<CertificationItem> items = new ArrayList<>();
        CertificationItem item1 = new CertificationItem();
        item1.setName("Rahul - Admin Role");
        CertificationItem item2 = new CertificationItem();
        item2.setName("Priya - SAP Read Access");
        items.add(item1);
        items.add(item2);
        campaign.setItems(items);

        // 🔹 7. Set Actions
        List<String> actions = new ArrayList<>();
        actions.add("Approve");
        actions.add("Revoke");
        actions.add("Delegate");
        campaign.setActions(actions);

        // 🔹 8. Set Mitigations
        List<String> mitigations = new ArrayList<>();
        mitigations.add("Manager Approval");
        campaign.setMitigations(mitigations);

        // 🔹 9. Set Applications
        List<String> applications = new ArrayList<>();
        applications.add("SAP");
        applications.add("Active Directory");
        campaign.setApplications(applications);

        // 🔹 10. Set Roles
        List<String> roles = new ArrayList<>();
        roles.add("Admin");
        roles.add("Finance");
        campaign.setRoles(roles);

        // 🔹 11. Set Entitlements
        List<String> entitlements = new ArrayList<>();
        entitlements.add("Read");
        entitlements.add("Write");
        campaign.setEntitlements(entitlements);

        // 🔹 12. Save to DB
        context.saveObject(campaign);
        context.commitTransaction();

        System.out.println("✅ AccessReview saved in DB successfully!");
    }
}

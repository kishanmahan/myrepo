<!DOCTYPE Rule PUBLIC "sailpoint.dtd" "sailpoint.dtd">
<Rule language="beanshell" name="Custom Multi-Attribute Correlation Rule" type="Correlation">
  <Description>
    Ye rule account ko Identity se correlate karta hai using EmployeeID ya Email.
  </Description>
  <Source>
  import sailpoint.object.*;
  import sailpoint.api.IdentityService;
  import sailpoint.tools.Util;
  import java.util.HashMap;
  import java.util.Map;
  import org.apache.commons.logging.Log;
  import org.apache.commons.logging.LogFactory;

  // 1. Logging Setup
  Log logger = LogFactory.getLog("custom.rule.correlation");
  Map returnMap = new HashMap();

  try {
      // Account attributes jo source system se aa rahe hain
      String accountEmpId = account.getStringAttribute("EMPLOYEE_NUMBER");
      String accountEmail = account.getStringAttribute("MAIL");

      logger.debug("Correlating account: " + account.getNativeIdentity());

      // 2. Logic A: Employee ID se Match karna (Primary Match)
      if (Util.isNotNullOrEmpty(accountEmpId)) {
          QueryOptions qo = new QueryOptions();
          qo.addFilter(Filter.eq("name", accountEmpId)); // Identity ka 'name' aksar EmpID hota hai
          
          int count = context.countObjects(Identity.class, qo);
          if (count == 1) {
              Identity match = context.getUniqueObject(Identity.class, qo);
              if (match != null) {
                  returnMap.put("identity", match);
                  logger.info("Match mil gaya via EmployeeID: " + accountEmpId);
                  return returnMap;
              }
          }
      }

      // 3. Logic B: Email se Match karna (Secondary Match - Fallback)
      if (Util.isNotNullOrEmpty(accountEmail)) {
          QueryOptions qoEmail = new QueryOptions();
          qoEmail.addFilter(Filter.eq("email", accountEmail));
          
          int emailCount = context.countObjects(Identity.class, qoEmail);
          if (emailCount == 1) {
              Identity match = context.getUniqueObject(Identity.class, qoEmail);
              if (match != null) {
                  returnMap.put("identity", match);
                  logger.info("Match mil gaya via Email: " + accountEmail);
                  return returnMap;
              }
          } else if (emailCount > 1) {
              logger.warn("Duplicate Email mila: " + accountEmail + ". Manual correlation zaroori hai.");
          }
      }

  } catch (Exception e) {
      logger.error("Correlation Rule Error: " + e.getMessage());
  }

  // Agar koi match nahi mila, toh empty map return karega (Account Uncorrelated rahega)
  return returnMap;
  </Source>
</Rule>

//////////*
1. account Object Kaha Se Aa Raha Hai?
Jab aap Account Aggregation (ya Refresh) task chalate hain, toh SailPoint target application (jaise Active Directory, SAP, ya JDBC) se data fetch karta hai.
Aggregation ke dauran, har ek entry (row/record) ek ResourceObject banti hai.
SailPoint us ResourceObject ko account variable ke naam se aapke Correlation Rule ke andar "inject" kar deta hai.
Isliye aapko code mein Account account = ... likhne ki zaroorat nahi padti, wo pehle se waha implicit object ke roop mein maujood hota hai.

2. Kis Source Se Aa Raha Hai?
Ye us Application source se aata hai jiske liye aapne rule likha hai.
Agar aap Active Directory aggregate kar rahe hain, toh account.getStringAttribute("distinguishedName") kaam karega.
Agar aap Database (JDBC) source use kar rahe hain, toh account.getStringAttribute("EMPLOYEE_ID") (jo aapne SQL query mein column name diya hoga) waha se aayega.

3. Ek Advanced Example (Logic Clear karne ke liye)
Aapne jo code pucha, usme ye data flow aise kaam karta hai:
Java
// Ye 'account' object SailPoint ne ResourceObject se convert karke diya hai
String accountEmpId = account.getStringAttribute("EMPLOYEE_NUMBER"); 
// 'EMPLOYEE_NUMBER' wahi attribute hona chahiye jo Application Schema mein 'Identity Attribute' ya 'Attribute' define hai.
Correlation Rule ke Important Implicit Objects:
Jab ye rule chalta hai, toh SailPoint aapko ye 4 cheezein deta hai:-
context: SailPoint API se baat karne ke liye (SailPointContext).
account: Target system ka wo account jo abhi process ho raha hai (Link object ka raw form).
application: Jis application (source) se data aa raha hai uski details.
environment: Current environment ki settings ka map.

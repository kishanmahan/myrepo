()Workflow ka matlab hai – ek predefined sequence of tasks jo automatic ya semi-automatic chalti hai 
jab koi event trigger hota hai. 😎
workflow decide karta hai ki kaunsa kaam kab aur kaise hoga. Enterprise me ye Automation ka backbone hai.

2️⃣ Types of Workflows in SailPoint:-
Type	                                Description
Provisioning Workflow-------User ko access assign karna, roles add/remove karna
Deprovisioning Workflow-----User ke access remove karna (employee leave hone par)
Reconciliation Workflow-----Application accounts ko SailPoint ke data ke saath sync karna
Access Request Workflow-----User apne liye new access request karta hai → approval chain follow hota hai
Policy Violation Workflow---Policy breach detect hone par alert, remediation ya approval trigger hota hai

4️⃣ Simple Example in SailPoint:-
import sailpoint.api.SailPointContext;
import sailpoint.object.Workflow;
import sailpoint.object.Identity;
import sailpoint.object.ProvisioningPlan;
import sailpoint.object.Link;
import sailpoint.api.ProvisioningProject;
import sailpoint.object.Application;
import java.util.List;

public class LeastPrivilegeExample {

    public static void main(String[] args) throws Exception {

        // 1️⃣ Context create / fetch
        SailPointContext context = SailPointFactory.getCurrentContext(); // Assume context setup

        // 2️⃣ Workflow fetch
        Workflow workflow = context.getObject(Workflow.class, "LeastPrivilegeWorkflow");

        // 3️⃣ Identity fetch
        Identity user = context.getObject(Identity.class, "john.doe");

        System.out.println("Starting Least Privilege Workflow for: " + user.getName());

        // 4️⃣ Step 1: Check current access
        System.out.println("Step 1: Checking current access...");
        workflow.executeStep("CheckAccess", user);

        // 5️⃣ Step 2: Identify extra access / roles
        System.out.println("Step 2: Removing unnecessary roles...");
        workflow.executeStep("DeprovisionExtraAccess", user);

        // 6️⃣ Step 3: Assign required roles
        System.out.println("Step 3: Provisioning required access...");
        workflow.executeStep("ProvisionRequiredAccess", user);

        // Optional: You can also manually create a ProvisioningPlan
        ProvisioningPlan plan = new ProvisioningPlan();
        plan.setIdentity(user);

        // Example: Remove extra access (links)
        List<Link> currentLinks = user.getLinks();
        for (Link link : currentLinks) {
            if (link.getApplicationName().equals("OldSystem")) {
                plan.add(DeleteLink(link));
            }
        }

        // Example: Assign required roles
        Application salesforceApp = context.getObject(Application.class, "Salesforce");
        plan.add(ProvisioningPlan.createAssignRolePlan(salesforceApp, user, "FinanceRole"));

        // Execute provisioning plan
        ProvisioningProject project = new ProvisioningProject(plan, "LeastPrivilegeProject");
        context.provision(project);

        // 7️⃣ Step 4: Notify manager
        System.out.println("Step 4: Sending notification...");
        workflow.executeStep("NotifyManager", user);

        System.out.println("Least Privilege Workflow completed for: " + user.getName());
    }

    // Dummy method for deleting link
    private static ProvisioningPlan.DeleteLink DeleteLink(Link link) {
        ProvisioningPlan.DeleteLink dl = new ProvisioningPlan.DeleteLink();
        dl.setLink(link);
        return dl;
    }
}

()System Property kya hai?
System Property ek key-value pair hoti hai jo system ya application ke liye configuration settings store karti hai.
Key: Property ka naam (jaise defaultPassword)
Value: Property ka actual value (jaise Welcome@123)
Ye properties program me kahin bhi access ho sakti hain, jaise workflow, rules, ya provisioning me.

1️⃣ System ka matlab:-
Ye ek computer ya application environment hota hai jahan program run hota hai.
Isme include hota hai:-
Server: Jahan application host hoti hai (jaise web server, SailPoint server).
Operating System: Windows, Linux, etc.
Application/Software: Jaise SailPoint IdentityNow ya IdentityIQ.

import sailpoint.api.SailPointContext;
import sailpoint.object.Identity;
import sailpoint.workflow.WorkflowContext;
import sailpoint.object.TaskResult;
import java.util.Arrays;
import java.util.List;
import java.util.Map;

public class WorkflowExample {

    public static void main(String[] args) throws Exception {
        
        // Assume we already have WorkflowContext aur SailPointContext
        WorkflowContext context = WorkflowContext.getCurrentWorkflowContext();
        SailPointContext sailpointContext = context.getSailPointContext();

        // 1️⃣ Identity object fetch
        Identity identity = sailpointContext.getObjectByName(Identity.class, "john.doe");

        // 2️⃣ addLinksInformation
        List<String> links = Arrays.asList("AD:JohnD", "Salesforce:JDoe");
        context.addLinksInformation(identity, links);
        System.out.println("Links added for " + identity.getName() + ": " + links);

        // 3️⃣ calculateIdentityDifference
        Identity oldIdentity = sailpointContext.getObjectByName(Identity.class, "john.doe_old");
        Map<String, Object> diff = context.calculateIdentityDifference(oldIdentity, identity);
        System.out.println("Identity Differences: " + diff);

        // 4️⃣ scheduleWorkflowEvent
        context.scheduleWorkflowEvent("OnboardingWorkflow", "2026-03-19T09:00:00");
        System.out.println("Workflow 'OnboardingWorkflow' scheduled for 2026-03-19T09:00:00");

        // 5️⃣ scheduleRequest
        context.scheduleRequest("AccessReview", "2026-03-20T10:00:00");
        System.out.println("Request 'AccessReview' scheduled for 2026-03-20T10:00:00");

        // 6️⃣ launchTask
        TaskResult result = context.launchTask("RoleAssignmentTask");
        System.out.println("Task 'RoleAssignmentTask' launched, result: " + result.getStatus());
    }
}

✅ FULL REAL WORKFLOW STEP CODE (NON-CUSTOM)
import sailpoint.object.Identity;
import sailpoint.object.ProvisioningPlan;
import sailpoint.object.ProvisioningProject;
import sailpoint.workflow.WorkflowContext;

// context already available hota hai (WorkflowContext)

// 1️⃣ Identity fetch (workflow variable se)
Identity identity = (Identity) context.get("identity");

// Safety check (real workflow me hota hai)
if (identity == null) {
    throw new Exception("Identity not found in workflow context");
}

// 2️⃣ Form data ko store karo (UI se aaya data)
context.assimilateProvisioningForm(context.getArguments());

// 3️⃣ Provisioning Plan lo (system generate karta hai)
ProvisioningPlan plan = (ProvisioningPlan) context.get("plan");

if (plan == null) {
    throw new Exception("ProvisioningPlan not found in workflow context");
}

// 4️⃣ Approval form build karo
context.buildPlanApprovalForm(plan);

// 5️⃣ Role assignment activate karo
// (plan ke andar jo roles already defined hain wo apply honge)
context.activateRoleAssignment(identity);

// 6️⃣ Provisioning Project lo (plan se auto banta hai)
ProvisioningProject project = (ProvisioningProject) context.get("project");

if (project == null) {
    throw new Exception("ProvisioningProject not found in workflow context");
}

// 7️⃣ Final provisioning execute karo
context.provisionProject(project);

🔐 SailPoint IdentityIQ (IIQ) kya hai?
SailPoint IdentityIQ (IIQ) ek Identity Governance & Administration (IGA) tool hai.
Iska kaam hota hai:-
User ka access manage karna
Kaun kis system me kya permission rakhta hai, track karna
Compliance aur audit manage karna
Access request aur approval workflow chalana
Role based access control (RBAC) implement karna
👉 Ye mostly enterprise companies me use hota hai jahan hazaaro employees aur multiple applications hote hain.

()Compliance aur audit manage karna kya hai?
--Compliance ka matlab hota hai:-
Company ke rules, policies aur legal requirements ke hisaab se user ko sahi access dena.

🔥 SailPoint me Compliance ka kaam:-
✅ 1. Access Certification (Review):-
Manager ya Application Owner check karta hai:-
Kya user ko ye access milna chahiye?
Kya extra access toh nahi hai?
Example:-
Rahul ke paas Finance + HR dono ka access hai
System bolega → “Review karo”

✅ 2. Segregation of Duties (SoD):-
Ek user ko conflicting access nahi milna chahiye.
Example:-
❌ User:-
Payment approve kare
Aur payment release bhi kare
Ye fraud risk hai.
SailPoint detect karta hai:-
SoD violation 🚨

🛡️️⃣ Policy Enforcement (SailPoint me):-
👉 Matlab kya?
System automatically check karta hai ki company ke security rules follow ho rahe hain ya nahi.
🔹 Types of Policies:-
✅Segregation of Duties (SoD):-
Ek user ko conflicting access nahi milna chahiye.
Example:-
❌ User:-
Payment approve kare
Aur payment release bhi kare
Ye fraud risk hai.
SailPoint detect karta hai:-
SoD violation 🚨

✅ 2. Password Policy:-
Minimum length
Special characters
Expiry

✅ 3. Role Policy:-
User ko sirf approved roles hi milne chahiye.

🧠 1️⃣ Risk Scoring (SailPoint me)
👉 Simple matlab:-
Har user ka ek Risk Score hota hai jo batata hai ki wo user kitna security risk hai.
🔹 Risk Score kis base par banta hai?
1. Role Risk
Agar role sensitive hai (Admin, Finance etc.)

2. Entitlement Risk
Specific permission ka risk value

3. Application Risk
Critical system ka access (SAP, DB, Prod Server)

4. Policy Violations
SoD violation hai ya nahi

🔥 Example:-
Maan le:-
Access	      Risk Value
DB Admin	     70
Prod Server	     60
SoD Violation	 40
Total Risk Score = 170 → 🔴 High Risk

🎯 Real Use:-
High risk user ko:-
Priority certification me dala jata hai
Extra review hota hai
Escalation possible hota hai.

📋 2️⃣ Review (Certification Process):-
👉 Review kya hota hai?
Manager ya Application Owner check karta hai:-
“Kya is user ko ye access abhi bhi chahiye?”
Ye process periodic hota hai:-
Quarterly
Half yearly
Annual
🔄 Certification Flow:-
Campaign create hota hai
Users assign hote hain reviewer ko
Reviewer dashboard me list dekhta hai
Har access pe action leta hai.
---------------------------------------------------------------------
✅ 1️⃣ Approve (Certification me):-
👉 Kya hota hai?
Reviewer (Manager / App Owner) decide karta hai ki:
“Haan, is user ko ye access abhi bhi chahiye.”
🔹 System kya karta hai?
Decision store karta hai
Timestamp save karta hai
Reviewer ka naam save hota hai
Audit trail maintain hota hai
📌 Example:-
Kishan → DB Access
Manager dekhta hai → Kishan still backend pe kaam kar raha hai
→ Approve
Access unchanged rahega ✅

1️⃣ Escalation kya hota hai?
--Escalation ka matlab hai kisi task ko higher authority ya next level user ko bhejna, jab assigned user time par action nahi leta.
Example:-
Manager ko Access Certification approve karna tha.
Manager ne 5 din tak kuch action nahi liya.
System automatically us task ko Manager ke manager ya kisi aur admin ko bhej deta hai.
👉 Is process ko Escalation bolte hain.

Simple flow:-
Task assign hua
Time limit cross ho gayi
System task ko next level ko bhej deta hai

📌 Use case:-
Certification
Access request approval
Policy violation
--------------------------------------------------------------------------------------------------------------
2️⃣ Campaign Create kya hota hai?
Campaign ka matlab hota hai Access Review process.
Jab company sab users ke access ko review karna chahti hai, tab ek Certification Campaign banaya jata hai.
Campaign create karne ka matlab

Admin ek certification process start karta hai jisme:-
Kis users ka access review hoga
Kaun reviewer hoga
Deadline kya hogi

Example:-
Company check karna chahti hai:-
Kis employee ke paas extra access hai.
Admin ek Certification Campaign create karta hai.

Fir:-
Managers ko tasks milte hain

Wo review karte hain:-
✔ Keep Access
❌ Revoke Access
---------------------------------------------------------------------------------------------------------------
3️⃣ Priority Certification kya hota hai?
Priority Certification ka matlab hai high importance wala certification.
Iska use tab hota hai jab:-
Critical application ho
Security risk ho
Urgent review chahiye

Example:-
Application:-
Finance system
Production database
Agar kisi user ko extra access mil gaya hai, to admin Priority Certification create kar sakta hai.
Difference
Type	                     Meaning
Normal Certification------Normal access review
Priority Certification----Urgent / high risk access review
--------------------------------------------------------------------------------------------------------
1️⃣ Certification kya hota hai?
--Certification ka matlab hota hai user ke access ka review karna.
Company time-time par check karti hai ki:-
Kis user ke paas kaunsa access hai
Kya wo access abhi bhi required hai
Koi unnecessary access toh nahi
Reviewer decide karta hai:-
✔ Approve / Keep Access
❌ Revoke Access

Example:-
User Ravi ke paas:-
Finance Application access
Database access
Manager check karega ki Ravi ko ye access abhi bhi chahiye ya nahi.
------------------------------------------------------------------------------------------------------------
2️⃣ Access Request Approval kya hai?
Ye access lene ka process hota hai.
Flow:-
1️⃣ User portal me access request karta hai
2️⃣ Request Manager ya Application Owner ko jati hai
3️⃣ Wo approve ya reject karte hain
4️⃣ Approve hone par access mil jata hai

Example:-
Ravi ko Salesforce access chahiye.

Flow:-
User → Request
Manager → Approve
System → Access provide

3️⃣ Manager Certification kya hai?
Is certification me Manager apne team members ke access review karta hai.
Manager kya check karta hai
Team member ke paas kaunsa access hai
Kya wo access abhi bhi required hai

Example:-
Manager ke paas 10 employees hain.

Manager review karega:-
Ravi
Amit
Pooja

Aur decide karega:-
✔ Keep Access
❌ Remove Access
-----------------------------------------------------------------------------------------------------------
4️⃣ Application Owner Certification kya hai?
Is certification me Application Owner access review karta hai.
Application owner wo person hota hai jo application ka responsible hota hai.

Example:-
Application:-
SAP
Salesforce
Oracle DB

Application owner check karega:-
Kaun kaun user is application ko access kar raha hai.

Aur decide karega:-
✔ Keep access
❌ Revoke access
-------------------------------------------------------------------------------------------------------------
5️⃣ Role Certification kya hai?
Is certification me roles review kiye jate hain.
Role ka matlab hota hai multiple permissions ka group.

Example;-
Role = "HR Role"

Is role me permissions ho sakti hain:-
Employee data read
Employee data update

Certification me check hota hai:-
Kya role me unnecessary permission hai
Kya role abhi bhi valid hai.
--------------------------------------------------------------------------------------------------------------
6️⃣ Identity Certification kya hai?
Is certification me pure identity (user) ka review hota hai.

Reviewer check karta hai:-
User ke paas kaunsa access hai
Kya user abhi bhi company me kaam kar raha hai

Example:-
User:-
Ravi

Access:-
SAP
Database
Email

Reviewer decide karega:-
✔ Keep
❌ Revoke
---------------------------------------------------------------------------------------------------------------
3️⃣ Account Group Certification kya hai?
Isme groups ka review hota hai.

Example:-
Active Directory me group:-
HR_Group
Finance_Group
Admin_Group

Reviewer check karta hai:-
Kaun kaun user group me hai.
Aur decide karta hai:-
✔ Keep
❌ Remove
------------------------------------------------------------------------------------------------------------
1️⃣ Entitlement Certification kya hai?
Entitlement ka matlab hota hai specific permission ya privilege jo kisi application me user ko diya jata hai.

Example:-
Application: Active Directory
Permissions / Entitlements:-
Read
Write
Admin
HR_Group
Finance_Group

Entitlement Certification me reviewer check karta hai:-
Kya user ko ye permission chahiye?
Kya permission remove karni chahiye?

Certification me options:-
Reviewer ke paas usually ye options hote hain:-
✔ Approve / Keep → Access same rahega
❌ Revoke → Access remove ho jayega

Example:-
User: Ravi
Access:-
SAP_Read
SAP_Admin

Manager bole:-
✔ SAP_Read keep
❌ SAP_Admin revoke

5️⃣ Provisioning kya hai?
Provisioning ka matlab hai user ko access dena ya remove karna.
Provisioning ka matlab hota hai application me user ka access create, update ya remove karna.
Ye kaam usually access request approve hone ke baad automatically hota hai.
Example:-
User ne request ki:-
SAP access

Approval ke baad:-
SailPoint automatically SAP me account create karega.

Provisioning types;-
Create account
Enable account
Disable account
Add role
Remove role
-----------------------------------------
1. Create Account kya hai?
Isme naya user account create hota hai.

Example:-
Employee company join karta hai.

System automatically create karega:-
Email account
Active Directory account
Application account

Example:-
Username: ravi.kumar
Application: SAP
Action: Account Create
---------------------------------------
2. Enable Account kya karta hai?
Agar account disabled tha to usko fir se activate kar diya jata hai.
Example:-
User leave par gaya tha aur account disable ho gaya tha.
Ab wapas aaya:-
➡ Account enable ho jayega.
-----------------------------------
3. Disable Account kya hai?
Isme user ka account temporarily disable kar diya jata hai.
Example;-
Employee:-
Suspended
Long leave
Security issue
Tab account disable ho jata hai.
User login nahi kar payega.
-------------------------------
4. Add Role kya hai?
Isme user ko ek role assign kiya jata hai.
Role ek permissions ka group hota hai.
Example:-
Role = HR Role
Permissions:-
Employee read
Employee update
Agar user ko HR role milta hai to automatically ye permissions mil jayengi.
-----------------------------------
5. Remove Role kya hai?
Isme user se role remove kar diya jata hai.
Example:-
User department change karta hai:-
HR → Sales
System karega:-
❌ Remove HR Role
✔ Add Sales Role
--------------------------------------

     ↓

     ↓
Provisioning Engine
     ↓
Application Update

❌ 2️⃣ Revoke (Certification me):-
👉 Kya hota hai?
Reviewer decide karta hai:-
“Ab is user ko ye access nahi chahiye.”
🔹 System kya karta hai?
Revocation task create hota hai
Provisioning engine trigger hota hai
Application se access remove hota hai
Audit log create hota hai

📌 Example:-

Old Admin role ab required nahi
→ Manager click karega Revoke

System:-
Access remove karega
Record store karega
Compliance proof ready karega

Kis basis pe action liya gaya

📌 Example Comments:-
Approve case:-
“Access required for ongoing production support.”
Revoke case:-
“User moved to HR team, DB access no longer needed.”

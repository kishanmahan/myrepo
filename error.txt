()Naya Code (Fixed & Safe ):-
context.search(Identity.class, qo, "name")
Isse sirf username (String) load ho raha hai, jo memory me 100 guna halka hai.

()Iterator ko sirf naam fetch karne ke liye use kiya aur fauran close (flushIterator) kar diya, workflow chalne se pehle.

()Pure Java Memory List (for (String name : identityNames)) par loop chal raha hai.

2. Naya Code (Jo fully safe hai):-
Java
// Step 1: Ek khali paper (List) liya
List identityNames = new ArrayList();

try {
    it = context.search(Identity.class, qo, "name"); // 🚀 Sirf Name uthaya, bohot fast
    while (it.hasNext()) {
        Object[] row = (Object[]) it.next();
        identityNames.add((String) row[0]); // List me naam likh liya
    }
} finally {
    if (it != null) {
        Util.flushIterator(it); // 🚀 Pehle hi Database connection ko flush aur close kar diya! DB free!
    }
}

// Step 2: Ab database se koi lena-dena nahi hai, list par aaram se workflow launch karo
for (String identityName : identityNames) {
    // 🚀 Ab ye line jitna chahe background me DB utilize kare, purana cursor block nahi hoga
    workflower.launch(wf, "WF-" + identityName + "-" + System.currentTimeMillis(), args);
}
